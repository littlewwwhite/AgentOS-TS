# 剧本格式规范

本文档定义 AgentOS novel-to-script skill 使用的标准中文微短剧场记格式。Phase 2（写作阶段）生成的所有剧本必须严格遵循本规范，Phase 3（解析提取阶段）依据本规范将文本解析为结构化 Schema 对象。

---

## 一、格式语法

### 1.1 集标记

每集以独立行标记：

```
第{N}集
```

N 为阿拉伯数字。集标记行独占一行，前后空行分隔。

### 1.2 场次头

每个场次以场次头行开头：

```
{ep}-{scene} {时间} {内/外} {地点}
```

| 字段 | 说明 |
|------|------|
| `{ep}-{scene}` | 集号-场次号，场次号按集内递增（1-1, 1-2, ..., 2-1, 2-2, ...） |
| `{时间}` | 时间词：日 / 夜 / 清晨 / 黄昏 / 午后 / 凌晨 |
| `{内/外}` | 只能是"内"或"外"之一 |
| `{地点}` | 具体地点名称 |

### 1.3 人物行

场次头下一行为人物行，列出本场所有在场角色：

```
人物：角色A、角色B、角色C
```

- 🔴 每场必须有人物行
- 在场即标注，即使不说话
- **造型状态标注**：同一 Actor 的服装/造型与默认外观不同时，紧跟 Actor 名称加 `【状态】`
  - 格式：`人物：张三【战甲】、李四、王五【婚纱】`
  - 无标注 = default 外观
  - 状态仅指**服装/造型变化**（如【战甲】【婚纱】【囚服】【便服】），不包含年龄/时间线变化
  - 跨时间线角色（幼年/老年）应在 catalog 中注册为独立 Actor，直接用不同名称区分

### 1.4 六类行

人物行之后，正文由以下六类行构成：

| 行类型 | 语法 | 说明 |
|--------|------|------|
| 动作行 | `▲描述内容` | 画面描述、人物动作、环境描写；以 `▲` 开头 |
| 对白行 | `角色名（情绪）：台词` | 情绪标注在角色名与冒号之间的括号内 |
| 对白行（无情绪） | `角色名：台词` | 省略情绪括号 |
| OS 行 | `角色名（OS）：内容` | 角色内心独白 |
| 字幕/系统行 | `【字幕：内容】` 或 `【系统提示：内容】` | 叠加文字信息 |
| 旁白行 | `旁白：内容` | 无角色归属的叙述 |

### 1.5 场次分隔规则

- 场次之间空**一行**
- 集标记与首个场次之间可不空行

## 二、完整示例

```
第一集
1-1 日 内 觉醒大厅
人物：楚凡、林雪、赵天昊、老师、同学若干
【字幕：异能觉醒时代】
▲聚光灯下，巨大的能量水晶闪烁着耀眼的光芒。全场肃静。
老师（高声宣布）：恭喜林雪同学，觉醒A级异能【碧涛】！
▲能量水晶投射出蓝色的A字，水波状的光环扩散开来。
▲同学们爆发出热烈的掌声和惊叹。林雪站在台上，神情高傲。
同学A（羡慕）：A级！不愧是林雪学姐！
老师（看向楚凡）：下一位，楚凡！我们学院的希望！
▲楚凡走上台，将手按在水晶上。
▲水晶暗淡无光，沉寂了数秒后，勉强浮现出一个灰色的、摇摇欲坠的"F"。
【字幕：F级异能【按摩】】
▲全场先是死一般的寂静，随即爆发出哄堂大笑。
赵天昊（夸张大笑）：按摩？哈哈哈哈！楚大天才，以后去澡堂我一定找你！
▲楚凡脸色煞白，难以置信地看着自己的手。
▲林雪径直来到楚凡面前，眼神冰冷。
林雪（冷漠）：我们之间，到此为止了。
▲林雪将一枚刻着"凡"字的玉佩，冷漠地塞进楚凡手中。
林雪（不带感情）：我林雪的男人，绝不能是废物。
▲楚凡的智能终端突然响起急促的通讯声。
▲一道严厉的虚拟投影出现在他面前，是家族长老。
家族长老（通讯，怒不可遏）：楚凡，你这个废物，家族的脸都被你丢尽了！三日之内，将所有资源折算成资金还回来！否则后果自负！
▲通讯切断，长老的投影消失。

1-2 夜 外 学院街道
人物：楚凡
▲倾盆大雨，楚凡失魂落魄地走在街上，全身湿透，手中紧紧攥着那枚冰冷的玉佩。
楚凡（OS）：天才？废物？呵呵……一夜之间，什么都没了……
▲楚凡停下脚步，眼中满是绝望，缓缓闭上眼睛。
▲一道冰冷的机械音在他脑海中响起。
【系统提示：检测到S级异能失控，位置：学院西三号巷。】
【系统提示：宿主可通过目标皮肤接触，吸收目标失控能量。】
▲楚凡猛地睁开眼，脸上写满了震惊。
```

## 三、Phase 3 解析规则

Phase 3 使用确定性解析器（`mcp__script__parse_script`）从剧本中提取结构化数据。

### 3.1 行类型识别

| 行类型 | 识别规则 | 解析用途 |
|--------|----------|---------|
| 场次头 | 匹配 `{ep}-{scene} {时间} {内/外} {地点}` | 构建 scene 对象 |
| 人物行 | 以 `人物：` 开头 | 提取角色名 → actor_ids |
| 对白行 | `角色名（情绪）：台词` 或 `角色名：台词` | 提取角色名 |
| OS 行 | `角色名（OS）：内容` | 提取角色名 |
| 动作行 | 以 `▲` 开头 | 不解析（下游 skill 处理） |
| 字幕/系统行 | `【字幕：内容】` 或 `【系统提示：内容】` | 忽略 |
| 旁白行 | `旁白：内容` | 忽略 |

### 3.2 角色提取规则

- 从 `人物：` 行和对白行（含 OS）收集唯一角色名
- 排除群演描述（如"同学若干""路人×3"）
- ID 按首次出场顺序分配：act_001, act_002...

### 3.3 场景提取规则

- 从场次头解析：episode_id, scene_id, environment (space + time), locations
- `actors` 来自该场的人物行 + 对白行中出现的角色
- scene_id 带集前缀：ep001_scn_001, ep001_scn_002, ..., ep002_scn_001, ...
