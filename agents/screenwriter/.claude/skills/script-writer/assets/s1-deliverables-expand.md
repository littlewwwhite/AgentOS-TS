# S1 原文分析与灵感提取 — 交付物模板

> 本文件定义 S1 阶段所有交付物的标准输出格式。写作时按此模板填充具体内容。

---

## 交付物 ① 原文分析报告

```
[NTSV2-ITEM: 原文分析报告]
【原文分析报告】

Logline：[一句话概括核心故事]
题材标签：[自动提取，如"都市/重生/复仇"]
类型标签：[自动提取，如"虐恋/爽文/甜宠"]
核心梗：[故事核心创意，如"重生复仇""替身觉醒"]
核心期待感：[读者最期待看到的内容]
背景设定：[时代、地点、社会背景]
主线冲突：[核心矛盾一句话]

核心角色清单：
  - [角色名]：[身份定位] | [性格关键词] | [核心动机] | [弧光方向]
  - ...

配角清单：
  - [角色名]：[身份定位] | [功能定位] | [出场范围]
  - ...

关键事件节点：
  1. [事件名称]（[位置]）：[对主线的影响]
  2. ...

原文体量评估：
  总字数：[X 字]
  章节数：[X 章]
  信息密度：[高/中/低]
  叙事完整性：[完整/节选/片段]

建议集数：[基于原文体量和节拍密度的建议]
建议 S2 模式：[结构设计 / 结构扩写]

原作文风特征：
  叙事节奏：[描述]
  对白风格：[描述]
  情感密度：[描述]
  描写偏好：[描述]
  爽感结构：[描述]
[/NTSV2-ITEM]
```

## 交付物 ② 六维度画像

```
[NTSV2-ITEM: 六维度画像]
【六维度画像】

Identity（人物身份/性格）：
  关键词：[2-5 个关键词]
  文本依据：[提取依据概述]
  置信度：[明确/隐含/空白]

Relationship（人物关系）：
  关键词：[2-5 个关键词]
  文本依据：[提取依据概述]
  置信度：[明确/隐含/空白]

World（世界观设定）：
  关键词：[2-5 个关键词]
  文本依据：[提取依据概述]
  置信度：[明确/隐含/空白]

Hook（核心钩子）：
  关键词：[2-5 个关键词]
  文本依据：[提取依据概述]
  置信度：[明确/隐含/空白]

Vibe（风格调性）：
  关键词：[2-5 个关键词]
  文本依据：[提取依据概述]
  置信度：[明确/隐含/空白]

Goldfinger（金手指）：
  关键词：[2-5 个关键词 / 无金手指]
  文本依据：[提取依据概述]
  置信度：[明确/隐含/空白]

结构性标签识别结果：
  确定性结构：[结构名称]（命中信号：[列举]）
  叙事架构：[对应的分集/分弧模式]
  平台适配：[评估结果]
  装饰性标签归集：[剩余标签列表]

第一推动力：[情感推动力/剧情推动力]
推动力分析：[为什么是这个推动力，2-3 句话，含原文依据]
[/NTSV2-ITEM]
```

## 交付物 ③ IP不动点清单

```
[NTSV2-ITEM: IP不动点清单]
【IP 不动点清单】

第一推动力：[情感推动力/剧情推动力] — 全剧核心驱动逻辑不可更改
世界观：[核心世界规则/设定，不可更改]
主线冲突：[故事主要矛盾，不可更改]

角色核心设定：
  - [角色名]：[不可更改的核心设定，如身份/性格底色/能力]
  - ...

角色弧光：
  - [角色名]：[成长轨迹，从A到B的转变]
  - ...

关键关系锚点：[核心人物关系的起点和终点]
时间线锚点：[必须保留的关键事件节点，按时间顺序]
结局走向：[HE/BE/OE，及核心结局事件]
关键意象/道具：[贯穿全剧的标志性元素]
金手指规则：[金手指的能力边界和限制（如有）]

不动点刚性分级：
  - 刚性不动点（绝对不可改）：[列举]
  - 弹性不动点（可微调但不可推翻）：[列举]
  - 改编自由区（原文未限定，改编可自由发挥）：[列举]
[/NTSV2-ITEM]
```

## 交付物 ④ 扩写潜力评估报告

```
[NTSV2-ITEM: 扩写潜力评估报告]
【扩写潜力评估报告】

一、主线信息密度
  原文主线事件数：[X 个]
  目标集数：[X 集]
  事件缺口：[需补充约 X 个事件/节拍]
  密度评级：[充裕/适中/稀疏/严重不足]

二、角色深度空白
  [角色名]：
    原文深度：[深/中/浅/仅提及]
    扩写需求：[无需/轻度补充/中度扩展/需要重建]
    扩写方向：[具体建议]
  ...

三、世界观空白
  已建立设定：[列举]
  空白区域：
    - [空白1]：[描述 + 优先级]
    - ...

四、支线潜力
  原文已有支线：[列举 + 扩展潜力]
  原文暗示的潜在支线：[列举 + 可发展方向]
  建议新增支线：[列举 + 理由 + 预估集数贡献]

五、情感空间
  主CP情感线扩展空间：[评估]
  其他情感线扩展空间：[评估]

六、扩写总结
  整体扩写难度：[低/中/高]
  建议 S2 模式：[结构设计 / 结构扩写]
  扩写重点方向（按优先级）：
    1. [方向]：[说明 + 预估集数贡献]
    2. [方向]：[说明 + 预估集数贡献]
    3. [方向]：[说明 + 预估集数贡献]
  扩写风险提示：
    - [风险]：[描述 + 规避建议]
[/NTSV2-ITEM]
```

## 交付物 ⑤ 风格方案

```
[NTSV2-ITEM: 风格方案]
【风格方案】

第一推动力：[情感推动力/剧情推动力]

推荐主风格：[代号 + 名称，如"A. 虐心催泪流"]
推荐理由：[基于原文文风特征和六维度特征的分析]

推荐副风格（可选）：[代号 + 名称]
组合理由：[为什么需要副风格辅助]

建议主副比例：[如 主70% / 副30%]

原文文风适配评估：
  兼容度：[高度兼容/部分兼容/风格冲突]
  需调整方面：[如有]

核心风格特征预览：
  对白特点：[该风格的对白特征]
  节奏特点：[该风格的节奏特征]
  画面调性：[该风格的画面调性]
  适用题材：[该风格适用的题材]
[/NTSV2-ITEM]
```

> 风格选项详见 `resources/style-options.md`。

## 交付物 ⑥ 创作锚点清单

```

核心风格：[风格代号 + 名称] — 确定后全流水线锁定
叙事架构：[结构性标签确定的底层架构] — 不可更改

改编方针：
  S2 模式：[结构设计 / 结构扩写]
  扩写方向：[从扩写评估中确定的重点方向]
  保留优先级：[哪些原文元素必须保留，哪些可以改编]

角色改编锚点：
  - [角色名]：[改编中不可变的设定 + 可调整的方面]
  - ...

情感线锚点：
  - 主CP：[情感线的核心轨迹，不可更改]
  - 副线：[如有，核心走向]

结局锚点：[HE/BE/OE + 核心结局事件]
金手指锚点：[金手指的能力边界和限制（如有）]
[/NTSV2-ITEM]
```
