# 剧本格式规范

本文档定义标准中文微短剧场记格式。Phase 2（写作阶段）生成的所有剧本必须严格遵循本规范，Phase 3（解析提取阶段）依据本规范将文本解析为结构化 Schema 对象。

---

## 一、格式语法

### 1.1 集标记

每集以独立行标记：

```
第{N}集
第{N}集：{标题}
```

N 为阿拉伯数字（1, 2, 3...），🔴 禁止中文数字（❌ `第一集` → ✅ `第1集`）。标题可选。集标记行独占一行，前后空行分隔。

### 1.2 场次头

每个场次以场次头行开头：

```
{ep}-{scene} {时间} {内/外} {地点}
```

| 字段 | 说明 |
|------|------|
| `{ep}-{scene}` | 集号-场次号，场次号按集内递增（1-1, 1-2, ..., 2-1, 2-2, ...） |
| `{时间}` | 时间词：日 / 夜 / 晨 / 清晨 / 黄昏 / 午后 / 凌晨 / 夜晚 / 深夜 / 黎明 / 白天 / 傍晚 / 中午 |
| `{内/外}` | 只能是"内"或"外"之一 |
| `{地点}` | 具体地点名称，可带状态后缀如 `地铁站【灯光切换】` |

### 1.3 人物行

场次头下一行为人物行，列出本场所有在场角色，可内联状态标注：

```
人物：角色A【战甲】、角色B、角色C
```

- 🔴 每场必须有人物行
- 🔴 前缀**必须精确写 `人物：`**——❌ `登场人物：` ❌ `角色：` ❌ `出场人物：` 均不被解析器识别
- 在场即标注，即使不说话
- 🔴 角色名必须与 catalog.json 中 `name` 字段**完全一致**——禁止使用代称/简称/关系称呼。`bilingual: true` 时 catalog 用英文名，剧本也必须用英文名；`bilingual: false` 时统一中文名
- 🔴 状态用 `【状态名】` 紧跟角色名，如 `楚凡【战甲】`。无【】= 默认外观
- 🔴 状态名必须与 catalog.json 中 `states` 数组名称一致，未注册状态会被解析器忽略
- 🔴 禁止用圆括号标注状态：❌ `角色（状态）` ❌ `角色(状态)`
- 跨时间线角色（幼年/老年）应在 catalog 中注册为独立 Actor，直接用不同名称区分

### 1.4 道具行

场次涉及 catalog.json 中关键道具时，人物行之后写道具行：

```
道具：道具名A、道具名B
```

- 仅列出本场画面中出现或被角色使用/提及的关键道具
- 🔴 道具名必须与 catalog.json 中 `name` 字段**完全一致**
- 无关键道具的场次可省略

### 1.5 六类正文行

人物行（+ 道具行 + 状态行）之后，正文由以下六类行构成：

| 行类型 | 语法 | 说明 |
|--------|------|------|
| 动作行 | `▲描述内容` | 画面描述、人物动作、环境描写；以 `▲` 开头，动作点用 `→` 衔接 |
| 对白行 | `角色名（情绪）：台词` | 情绪标注在角色名与冒号之间的括号内 |
| 对白行（无情绪） | `角色名：台词` | 省略情绪括号 |
| OS 行 | `角色名（OS）：内容` | 角色内心独白 |
| 字幕/特殊发声体行 | `【字幕：内容】` / `【系统提示：内容】` / `【广播】：内容` | 字幕保留为文字层；系统/广播/面板提示等可提为特殊 speaker |
| 旁白行 | `旁白：内容` | 无角色归属的叙述 |

### 1.5.1 台词语言规则（条件触发）

由 `design.json.bilingual` 字段控制，Phase 1 自动检测原文是否含中英双语台词。

**`bilingual: true`（原文含中英双语）：**

🔴 剧本只保留英文台词，删除中文行。格式：

```
角色名（情绪）：English dialogue only.
```

示例：
```
Fat Matron（sneering）：You lowly, ruined swine! Do you think you're still some pampered lady?
Rosalind（restrained）：I will wash it again. Take your foot off the clothes.
Cyrus（cold）：You, look up.
```

- 🔴 情绪标注使用英文
- 🔴 画面内文字或特殊发声体文案（`【字幕：】`/`【系统提示：】`/`【广播】：`）也使用英文
- 🔴 英文台词必须取自原文，禁止自行翻译或捏造
- 🔴 角色名、人物行、动作行中的角色引用均使用 catalog 中的英文 `name`

**`bilingual: false`（原文单语）：**

保留原文语言不变，禁止自行添加翻译。角色名使用 catalog 中的中文 `name`。

### 1.6 场次分隔规则

- 场次之间空**一行**
- 集标记与首个场次之间可不空行

### 1.7 每场结构顺序

```
场次头
人物行（必填，可内联状态标注如 角色【状态】）
道具行（有关键道具时）
正文（动作行/对白行/OS行/字幕行/旁白行）
```

## 二、完整示例

```
第1集
1-1 日 内 觉醒大厅
人物：楚凡、林雪、赵天昊、老师、同学若干
道具：凡字玉佩
【字幕：异能觉醒时代】
▲宽阔圆形大厅穹顶锥形白光直射地面 → 水晶悬浮旋转冷白光芒脉动着向外扩散 → 四周学员身着深蓝制服齐刷刷望向台上
老师（高声宣布）：恭喜林雪同学，觉醒A级异能【碧涛】！
▲水晶投射蓝色"A"字 → 水波光环向外炸开 → 全场掌声爆发 → 林雪站台上下巴微抬嘴角含笑
同学A（羡慕）：A级！不愧是林雪学姐！
▲老师转向楚凡 → 手指引导楚凡走上台
老师（看向楚凡）：下一位，楚凡！我们学院的希望！
▲楚凡深吸一口气挺直脊背大步走上测试台 → 右手缓缓按上水晶手指因紧张微微发颤
▲水晶白光骤然暗淡成灰映得楚凡脸色惨白 → 灰色"F"字摇摇欲坠地浮出
【字幕：F级异能【按摩】】
▲周围学员先是呆愣 → 笑声从四面八方炸开有人笑弯腰拍着大腿
赵天昊（夸张大笑）：按摩？哈哈！楚大天才，以后去澡堂我一定找你！
▲楚凡脸色煞白双手从水晶缓缓收回 → 低头看着自己颤抖的指尖
▲林雪穿过人群脚步利落眼神从惊愕一点点冻成冰 → 径直走到楚凡面前
林雪（冷漠）：我们之间，到此为止了。
▲林雪将凡字玉佩塞进楚凡手中 → 转身离去头也不回
林雪（不带感情）：我林雪的男人，绝不能是废物。
▲楚凡智能终端突然响起急促通讯声 → 手腕上方弹出虚拟投影 → 家族长老愤怒的面容浮现
家族长老（通讯，怒不可遏）：楚凡，你这个废物，家族的脸都被你丢尽了！三日之内，将所有资源折算成资金还回来！否则后果自负！
▲通讯切断投影碎裂消散成光点 → 空旷测试台上只剩楚凡一人的背影

1-2 夜 外 学院街道
人物：楚凡
道具：凡字玉佩
▲倾盆大雨猛烈拍打路面 → 楚凡失魂落魄地走在街上全身湿透 → 楚凡右手紧紧攥着那枚冰冷的玉佩
楚凡（OS）：天才？废物？呵呵……一夜之间，什么都没了……
▲楚凡停下脚步眼中满是绝望 → 楚凡缓缓闭上眼睛
▲一道冰冷的机械音在楚凡脑海中响起 → 楚凡猛地睁开眼脸上写满震惊
【系统提示：检测到S级异能失控，位置：学院西三号巷。】
【系统提示：宿主可通过目标皮肤接触，吸收目标失控能量。】
【广播】：学院广播即将开始全域播报。
```

🔴 **注意以上示例中：**
- 集标记用阿拉伯数字 `第1集` 而非 `第一集`
- 每场首行是 `人物：`，前缀精确无误
- 动作行用 `→` 衔接动作点，不用句号分句
- 动作行中无人称代词（他/她），全部用角色名
- 每条对白上方都有 `▲` 动作行
- 无任何 markdown 标记（`#` `##` `---` `**`）

## 三、Phase 3 解析规则

Phase 3 使用确定性解析器（`python3 ${CLAUDE_SKILL_DIR}/scripts/parse_script.py`）从剧本中提取结构化数据。

### 3.1 行类型识别

| 行类型 | 识别规则 | 解析用途 |
|--------|----------|---------|
| 场次头 | 匹配 `{ep}-{scene} {时间} {内/外} {地点}` | 构建 scene 对象 |
| 人物行 | 以 `人物：` 开头 | 提取角色名 → actor_ids + 内联状态 `角色【状态】` → state_ids |
| 道具行 | 以 `道具：` 开头 | 提取道具名 → prop_ids |
| 状态行 | 以 `状态：` 开头（向后兼容，新剧本不使用） | 提取角色状态标注 → state_ids |
| 对白行 | `角色名（情绪）：台词` 或 `角色名：台词` | 提取角色名 |
| OS 行 | `角色名（OS）：内容` | 提取角色名 |
| 动作行 | 以 `▲` 开头 | 匹配 catalog 角色名 → actor_ids（容错补充） |
| 字幕/特殊发声体行 | `【字幕：内容】` / `【系统提示：内容】` / `【广播】：内容` | `字幕` 提取为 `sfx`；其余系统/广播/机械音/面板提示等提取为特殊 speaker 的 `dialogue` |
| 旁白行 | `旁白：内容` | 忽略 |

### 3.2 角色提取规则

- 从 `人物：` 行、对白行（含 OS）、动作行（匹配 catalog 角色名）收集唯一角色名
- 系统、广播、机械音、面板提示、器灵等**非人类但会稳定发声的主体**，也会作为 actor 提取，供下游声音资产和音色一致性使用
- 排除群演描述（如"同学若干""路人×3"）
- ID 按首次出场顺序分配：act_001, act_002...

### 3.3 场景提取规则

- 从场次头解析：episode_id, scene_id, environment (space + time), locations
- `actors` 来自该场的人物行（含内联状态）+ 对白行中出现的角色
- `props` 来自该场的道具行
- scene_id 格式：scn_001, scn_002...（每集重新计数）
