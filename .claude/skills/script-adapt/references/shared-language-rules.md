# 跨流水线语言规则

本文件是 `script-adapt` 与 `script-writer` 共用的**语言一致性检查清单**。它只定义跨 skill 的共识，不重复完整的写作技法或场记格式。

## 1. 项目语言先锁定，再全程一致

- 默认输出语言为**中文简体**
- 若用户、上游原文或前置设计已经明确目标语言，则从 S1 / Phase A 起锁定该语言，不得中途切换
- 单个项目内，角色名、专有名词、对白、OS、字幕/系统提示、旁白，必须遵循同一套语言策略

## 2. 角色名与专有名词必须全链路一致

- 剧本中的角色名必须与上游正式名录保持一致：
  - `script-adapt` 以 `catalog.json` 为准
  - `script-writer` 以 `anchor.md` / S4 角色卡为准
- 不得在剧本阶段临时改名、缩写、换称呼后又要求解析器或下游 prompt 猜测

## 3. 不得擅自翻译、补写或混写

- 如果上游源文或设计没有要求切换语言，就保持原定语言输出
- 不得为了“更自然”临时把中文对白改英文，或把英文对白改中文
- 不得把同一条对白写成双语并列，除非上游设定明确要求双语呈现

## 4. 特殊发声体与画面内文字也遵循同一语言策略

- `【字幕：】`、`【系统提示：】`、`【广播】：` 等特殊发声体，语言规则与主剧本文本一致
- 若项目要求可见文字内容，必须与该项目的目标语言一致；若下游视频阶段要求去文字，则在分镜 / 视频阶段移除可见文字，不在 SCRIPT 阶段混入第二套语言

## 5. 技法与格式的唯一事实源

- 场记格式唯一事实源：`.claude/skills/script-adapt/references/script-format.md`
- `script-adapt` 的原文保真与改编写作约束：`writing-rules.md`
- `script-writer` 的原创 / 扩写写作约束：`.claude/skills/script-writer/references/writing-rules.md`

## 6. 条件触发规则

### 对 `script-adapt`

- 若 `design.json.bilingual = true`，严格遵循 `script-format.md` 中“台词语言规则（条件触发）”
- 若 `design.json.bilingual = false`，保留原文语言，不得自行添加翻译
- 改编阶段不得编造原文不存在的对白

### 对 `script-writer`

- 若用户未指定特殊语言，默认以中文简体完成 S1-S9
- 若项目明确是英文或其他语种项目，则从前置设计开始锁定，并在全部阶段保持一致
- 若为扩写模式，语言跟随上游 source 与项目设定，不得在 S7 / S8 临时切换
