# 写作核心规则

> 以下规则是 Phase 2（分集写作）的**单一真相源**，每一行剧本必须遵循。动作行以 `▲` 开头，写作时用「叙事/演绎/空镜」三类思维模式指导取舍。

> **优先级标记：**
> - 🔴 **MUST**：违反即判定不合格，必须返工
> - 🟡 **SHOULD**：强烈建议遵循，特殊情况可灵活处理
> - 🟢 **MAY**：可选增强，视场景决定

---

## 零、台词铁律（条件触发）

> 由 `design.json.bilingual` 字段控制。Phase 1 自动检测原文是否含中英双语台词。

- 🔴 **`bilingual: true`**：剧本只保留英文台词，删除中文行；情绪标注使用英文；画面内文字（字幕/系统提示）使用英文
- 🔴 **`bilingual: false`**：保留原文语言不变，禁止自行添加翻译
- 🔴 **禁止捏造**：所有台词必须源自原文，禁止凭空发明原文不存在的对白
- 🔴 **禁止遗漏**：逐段比对原文，确认所有台词均已演绎，无跳过

## 一、对白规则

### 1.1 短句与节奏

- 🔴 短句（8–14 字），三句一转折
- 🟡 10–12 秒投一条新信息

### 1.2 对白功能标签（五类）

> 🔴 所有对白优先用以下五类功能标签驱动，每场至少命中两类：

- **事件**：让角色直接宣布决定/动作——"掌门之位，我辞了。"
- **回忆**：一句一画面，只说与当下选择相关的记忆——"他教我第一句是别怕。"
- **情绪**：用行为/指令句承载，少抽象形容——用动作表达而非直说"我很难过"
- **感情**：通过边界/让步/选择落地，不做空泛表白——"我喜欢你，不躲了。"
- **欲望**：明确目标与代价，要什么、立刻放弃什么——"我想赢一次。"

### 1.3 代言发声

🟡 当主角不宜开口时，由环境或旁人代为"落锤"。

**触发场景**：身份暴露风险、情绪崩溃无法言语、信息需第三方视角传递、权力关系不对等。

**代言方式**：
- **环境代言**：铃声/广播/系统提示——"系统提醒：欠款到期，现在扣款。"
- **旁人代言**：闺蜜/属下/小贩替主角发声

### 1.4 对白呈现格式（4 种 + 心声修饰符）

> 🔴 每条对白 = 1 个呈现格式 × 1–2 个功能标签。

```
角色对白：  角色名（情绪）：台词
内心独白：  角色名（OS）：独白内容
环境音：    【广播】：广播/系统语音内容
心声修饰：  台词[心声]隐秘心思         ← 附加在对白末尾
```

### 1.5 心声使用原则

- 🟢 一般少用，附在对话后面以提升情绪强度
- 心声是无法言说的隐秘心思，不能说出口的痛苦感情
- 示例："沈家只是配合公主演戏。[心声]这次定要沈家家破人亡"

### 1.6 句式调控

- 🔴 短小精悍的句子，强调情节紧张
- 🟡 简单通顺的句式，提高可读性
- 🟡 感叹句、反问句表达强烈情感
- 🟡 断句技巧，突出剧情转折
- 🟢 排比句式，增加文本节奏感

## 二、画面规则

### 2.1 画面类型（3 种写作思维）

> 🟡 建议比例：叙事思维 55% / 演绎思维 35% / 空镜思维 10%

在标准场记格式中，三类动作行均以 `▲` 开头，写作时以思维模式决定内容取舍。

**叙事思维 — 对白主导，画面承载对白**

画面为对白提供视觉容器——让观众知道"谁在说"、"在哪说"、"说的时候在做什么"。核心信息由对白传递，画面搭台。闪回/回忆用 `（回忆）/（现实）` 标注。

```
1-1 日 内 咖啡厅
人物：苏瑶、楚凡、夏语冰
▲两人隔着小圆桌对坐，苏瑶不停搅拌咖啡，勺子碰杯壁叮叮作响
苏瑶（低头）：那天晚上的事……你还记得吧。

▲（回忆）阴暗小巷中 → 楚凡撕开她的衣服 → 低吼：别动！
▲（现实）夏语冰擦去泪水 → 眼中闪过决然
```

**演绎思维 — 画面主导，动作即内容**

画面本身就在讲故事，是对白之间的表演段落——打斗拆招、情感爆发的肢体表现、关键行为的连续展开。对白缺席或仅作短句点缀。

```
▲楚凡侧身闪过直拳，反手扣住对手手腕 → 膝盖顶上腹部，对手弯腰闷哼 → 一拧手腕将对手摔倒，单膝压住后背
```

**空镜思维 — 无角色动作，环境与留白**

无角色核心动作，用于：① 场景转换 ② 情绪沉淀 ③ 环境特写 ④ 暗示回避

```
▲倾盆大雨 → 路灯孤影 → 水洼倒映破碎天空
```

> **类型判定**：这段内容的核心信息/体验由什么传递？
> - 对白/OS/VO 传递 → 叙事思维
> - 肢体动作/物理变化传递 → 演绎思维
> - 无角色动作 → 空镜思维

### 2.2 画面即容器原则

> 🔴 **核心法则：对白行紧邻上方的动作行，描述的是该对白播出时观众正在看到的画面，而非对白之前已经完成的动作。**

画面行与对白行构成**同一时间切片**——画面是视觉轨道，对白是声音轨道，二者同时播放。

**结构公式：**

```
说话时观众看到的画面（视觉轨道）           ← ▲ 动作行
角色名（情绪）：说话内容（声音轨道）        ← 对白行
↑ 这两行在同一时间段内同步播出
```

**三条硬规则：**

- 🔴 **有白必有画**：每一句对白（含 OS）的紧邻上方必须有 `▲` 动作行，该动作行是对白的视觉载体
- 🔴 **画随白走**：`▲` 动作行描述的动作/状态必须与对白**同时发生**——观众在听到这句话的同时看到这个画面
- 🔴 **前因后果分行**：若一段动作在对白之前完成，该动作必须独立为前置 `▲` 动作行，与对白的载体动作行分开

**典型结构模式：**

```
模式A：动作 → 说话（最常见）
▲角色完成某个动作序列               ← 说话前的动作（纯视觉，无对白）
▲角色说话时的表情/姿态/小动作       ← 说话时的画面（对白的视觉载体）
角色名（情绪）：对白               ← 与上行画面同步

模式B：边做边说
▲角色一边做某事 → 一边开口          ← 做和说同时发生
角色名（情绪）：对白               ← 与上行画面同步

模式C：纯动作段（无对白）
▲连续动作序列                       ← 无对白的表演段落
▲继续动作                           ← 仍无对白

模式D：反应 + 回应
▲A说完后，B的表情反应               ← B听完后的视觉反应
▲B开口回应时的姿态                  ← B说话时的画面
角色B（情绪）：回应对白            ← 与上行画面同步
```

**正误对比：**

```
❌ 错误：画面是"说话前"的动作，说话时没画面

▲沈斯淮抄起玻璃杯 → 砸在脚下 → 碎片四溅 → 一片扎进小腿
沈斯淮（暴怒）：就是因为那两个老不死的恢复了意识！
                 ↑ 4动作点=12秒已用完，说话时观众看什么？

✅ 正确：前因动作独立，说话时有专属画面

▲沈斯淮抄起玻璃杯 → 狠狠砸在她脚下 → 碎片四溅
▲他站在碎片中间居高临下 → 手指直指跪在地上的青音 → 咬牙切齿
沈斯淮（暴怒）：就是因为那两个老不死的恢复了意识！
                 ↑ 观众看到他指着她咬牙的画面，同时听到这句话
```

```
❌ 错误：纯对白无画面载体

宋青音（恳求）：阿淮，爸妈年纪大了，真的经不住……
                 ↑ 她说话时观众看到什么？不知道

✅ 正确：对白有说话时的画面

▲宋青音跪在地上仰头看他 → 双手攥紧裙摆 → 声音在发抖
宋青音（恳求）：阿淮，爸妈年纪大了，真的经不住……
                 ↑ 观众看到她跪着仰头、手攥裙摆的画面
```

```
❌ 错误：长画面后跟 OS，OS 播放时画面已演完

宋青音推开门 → 看见沈斯淮含冰块 → 喂进温宁口中 → 两人激吻 → 周围烟雾缭绕 → 酒瓶散落
宋青音（OS）：这种画面我见过很多次了。

✅ 正确：推门是前因动作，OS 的载体是"她看到的画面"

▲宋青音猛推开包厢门 → 门砸在墙上 → 灯光暧昧的包厢映入眼帘
▲沈斯淮半靠沙发含着冰块 → 低头喂进温宁口中 → 两人旁若无人地激吻
宋青音（OS）：这种画面我见过很多次了。可每次……都觉得刺眼。
               ↑ 观众看着两人激吻的画面，同时听到她的内心独白
```

**自检方法：**

1. 指着每一句对白问：**"说这句话时屏幕上在放什么？"** → 答案必须在紧邻上方的动作行中
2. 遮住所有对白，只看动作行序列——应是一段连贯无空白的视觉流
3. 遮住所有动作行，只看对白——应是一段独立成立的语音流

### 2.3 动作点规则

> 🔴 **动作点 = 一段约 3 秒的连贯动作序列**，描述角色在这 3 秒内的可见动作。允许简洁表达，不强制填满每一秒的动作密度——清晰传达画面意图即可，避免为凑密度而堆砌微表情或碎片动作。

- 🔴 动作行内用 `→` 衔接动作点，形成角色完整动线
- 🔴 每行 **≤ 5 个动作点**
- 🔴 行为逻辑必须连贯

**合格动作点的三个硬标准：**

🔴 **① 3 秒动作清晰可视**

这 3 秒内角色有可见的动作变化，画面意图清晰。允许简洁的动作描述，不必强制填满每一秒。

```
✅ 简洁有效："拿起杯子抿了一口"
✅ 详细亦可："端起茶杯凑到唇边，犹豫一瞬，浅浅抿了一口"
❌ 过于碎片："拿起杯子"（无动作指向，画面不完整）

✅ 简洁有效："猛地站起来双手撑桌"
❌ 过度堆砌："猛地推开椅子站起来，双手撑在桌面上，指尖把桌面压得发白"（微表情冗余）
```

🔴 **② 与相邻动作点无内容重叠**

前后两个动作点描述的是不同的 3 秒，不是同一动作的不同说法，也不是动作与其结果状态的拆分。

```
❌ 重叠："俯下身 → 与她面对面"（动作 + 结果，实为同一个 3 秒）
✅ 不同："俯下身凑到她面前、目光平视 → 伸手轻轻抬起她的下巴"

❌ 重叠："握紧拳头 → 手指攥得发白"（同一状态换个说法）
✅ 不同："双拳攥紧、指节发白 → 一拳砸在墙上，石灰簌簌掉落"
```

🔴 **③ 合并优先原则**

连续的微表情、情绪反应、同一角色小动作应合并为一个动作行，而非拆成多个 ▲ 行。优先保留推动剧情或传递关键信息的动作，删除纯装饰性的情绪反复。

```
❌ 拆分冗余（3 行）：
▲Hailey眼眶泛红
▲Hailey低下头攥紧裙摆
▲Hailey深吸一口气抬起头

✅ 合并为 1 行：
▲Hailey眼眶泛红，深吸一口气抬起头
```

判断标准：
- 同一角色连续 2 个以上纯情绪/微表情 ▲ 行 → 必须合并
- 合并后仍需保持主谓宾结构完整
- 对白载体行（紧邻对白上方的 ▲）不可被合并删除

🔴 **④ 每个动作点必须是完整的主谓宾结构，禁止使用人称代词**

每个 `→` 分隔的动作点必须有**明确主体（主语）+ 具体动作（谓语）+ 动作指向（宾语/补语）**，禁止写纯形容词、无主语的场景感叹、或抽象状态描述。主语可以是角色名、具体物体或环境元素，但必须明确写出。

🔴 **动作行（`▲` 行）中禁止使用人称代词（他/她/它/他们/她们）**，必须全部替换为角色名。每个 `→` 两侧的动作点都必须有独立的、明确的角色名主语，不依赖上文推断。

```
❌ 无主语/纯环境感叹："月光洒落"
✅ 主谓宾完整："月光透过窗棂，将楚凡的脸切成明暗两半"

❌ 形容词堆砌，无动作："衣衫褴褛"
✅ 主谓宾完整："Fat Matron俯身抓住Rosalind衣领，将Rosalind从盆边拖开"

❌ 抽象状态，无主体动作："空气凝固" / "四周一片死寂" / "无形的火花碰撞"
✅ 主谓宾完整："全场宾客屏住呼吸，无一人出声"

❌ 叙述评价，非动作："场面混乱"
✅ 主谓宾完整："侍卫蜂拥冲入大殿，兵器碰撞声响彻廊道"

❌ 无宾语，动作指向不明："楚凡走过去"
✅ 主谓宾完整："楚凡大步走到沈斯淮面前，一把抓住沈斯淮的衣领"

❌ 人称代词替代角色名（最常见错误）：
  "嘴角勾起一抹危险的弧度，缓步向她走去"（无主语 + 人称代词"她"）
  "她没有退缩，修长白指悄悄握紧"（代词"她"）
  "缓缓抽出侍卫腰间长剑"（无主语）
✅ 全部替换为角色名：
  "赛勒斯嘴角勾起一抹危险的弧度，缓步向罗莎琳德走去"
  "罗莎琳德没有退缩，修长白指悄悄握紧"
  "赛勒斯缓缓抽出侍卫腰间长剑，冰冷剑锋挑起罗莎琳德的下巴"
```

> **空镜行例外**：空镜思维段落（无角色核心动作，用于场景转换/情绪沉淀）允许以具体物体或自然元素为主语，但仍需写出可见的动态行为，而非静态形容。
> ```
> ❌ 空镜堆形容词："倾盆大雨 → 路灯孤影 → 破碎天空"
> ✅ 空镜有动态："大雨猛烈拍打路面 → 路灯光晕被雨水晕成一片 → 水洼倒映的天空随涟漪破碎"
> ```

**不合格动作点排除（更新）：**

| 类型 | 示例 | 修正 |
|------|------|------|
| 动作碎片（无画面指向） | "拿起杯子"（无后续） | 补出动作指向："拿起杯子抿了一口" |
| 结果状态（非新动作） | "面对面" | 合并到前一动作点 |
| 否定描述（无正向动作） | "没停下来" | 改写为可见动作："脚步踉跄但越跑越快" |
| 叙述评价 | "崴了脚也没停" | 改写或移入对白/OS |
| 换词重述 | "握紧拳头 → 手指攥得发白" | 合并为一句："双拳攥紧，指节发白" |
| 无主语场景感叹 | "月光洒落" / "空气凝固" | 补出主体："月光穿过云层，打在楚凡脸上" |
| 纯形容词/状态描述 | "衣衫褴褛" / "神情颓败" | 改写为动作："他扯了扯破烂衣角，低头不言" |
| 动作指向模糊 | "楚凡走过去" | 补出指向："楚凡走到窗前，手指抵上冰冷的玻璃" |
| 无施动主语的被动句 | "锈铁门被猛力撞开" | 补出施动者："近卫军将锈铁门猛力撞开" |
| 器物/身体部位独立做主语 | "冰冷剑锋挑起罗莎琳德下巴" | 改以人物为主语："赛勒斯以冰冷剑锋挑起罗莎琳德下巴" |

### 2.4 时空耦合模型

画面和对白共享同一段屏幕时间。画面提供时间容器，对白必须在容器内说完。

> 🔴 此模型与「画面即容器原则」联合执行：紧邻对白上方的动作行既是对白的视觉载体，也是对白的时间容器。

**基准换算：**

```
1 动作点 ≈ 3 秒画面时间
1 秒 ≈ 3 个字对白
∴ 1 动作点 ≈ 可容纳 9 字对白
```

**容量校验规则：**

```
对白字数 ≤ 紧邻动作行的动作点数 × 9

超出 → 拆分为多个时间单元，为对白增加反应/动作的画面承载
空余 → 留白作为情绪呼吸空间
纯对白 → 仍需至少 1 个动作点提供画面指导
```

**写作阶段执行要点：**

- 🔴 **一行动作配一行对白**是基本节奏单元，对白载体动作行动作点 ≥ 2（为对白提供足够的视觉载体时间）
- 🔴 **无对白的纯动作段落**独立成行，不与后续对白的载体动作行合并
- 🔴 写作阶段即执行时空耦合速算：写完每行对白后，立即验证「对白字数 ≤ 紧邻动作行动作点数 × 9」

**验证示例：**

```
▲楚凡终端响起急促通讯声 → 手腕上方弹出虚拟投影 → 家族长老愤怒的面容浮现
                                    3 动作点 = ~9秒 = 27字容量
家族长老（通讯，怒不可遏）：楚凡，你这个废物，家族的脸都被你丢尽了！
                                    21字 ≈ 7秒 ✅

▲楚凡面色剧变 → 长老投影逼近放大 → 楚凡双拳攥紧，指节发白
                                    3 动作点 = ~9秒 = 27字容量
家族长老（通讯，厉声）：三日之内，将所有资源折算成资金还回来！否则后果自负！
                                    22字 ≈ 7秒 ✅

▲通讯切断 → 投影碎裂消散成光点 → 空旷测试台上只剩楚凡一人的背影
                                    3 动作点 = ~9秒（留白，情绪沉淀）
```

### 2.5 画面内文字语言（条件触发）

> 由 `design.json.bilingual` 控制，仅 `bilingual: true` 时生效。

- 🔴 **`bilingual: true`**：`【字幕：】` 和 `【系统提示：】` 内容使用英文
- `bilingual: false`：保留原文语言

### 2.6 视觉叠层

🔴 字幕和系统提示独占一行：

```
【字幕：F级异能【按摩】】
【系统提示：检测到S级异能失控。】
```

### 2.7 卡点分隔

🟡 `（X卡）` 用于标记广告插入点、大幕间隔、重大转折前的呼吸空间。

```
（1卡）
```

### 2.8 动作行描述语言标准

> 🔴 动作行必须是**能有效指导 AI 生成画面的视觉描述**。

- 🔴 **具体可视化**：用可被"看见"的词代替抽象概念（❌"紧张" → ✅"双手攥紧裙摆，指节发白"）
- 🔴 **环境锚定**：首个动作行需建立场景的空间感、光线、色调、氛围
- 🔴 **动作明确**：遵循动作点规则，3 秒连贯动作流，密度充足、不留空档
- 🟡 **细节层次**：先整体空间→再人物位置→最后局部表情手势

### 2.9 场景一致性原则

- 🔴 同一场景内的动作行共享环境设定
- 🔴 场景首个动作行充分建立环境，后续可省略环境只写动作
- 🔴 切换场景必须通过新的场次编号（`{ep}-{scene}`）明确切分

### 2.10 角色引用原则

- 🔴 角色外观由锚点文档中的角色设定卡/配角速写卡统一定义
- 🔴 动作行通过角色名引用角色，着重呈现**动作、表情、姿态**变化
- 🔴 不在动作行重复描述角色固定外观

### 2.11 角色分层与在场标注

> 🔴 **在场必标注**：即使某角色全程不说话，只要剧情逻辑上 TA 应该在场，就必须出现在人物行中。场景不能只有说话的人——画面需要完整的人物关系空间。

**三级角色标注规则：**

- 🔴 **主角/配角**：直接写角色名，动作行可用角色名引用其动作和表情
- 🟡 **龙套**：有台词时用名称（如"路人甲"），无台词时用描述性称呼
- 🟡 **NPC**：仅在场景建立画面或群体画面中以群体形式出现（如"侍卫×2""围观路人"）

**人物行格式：**

```
人物：苏青鱼、庄晓蝶 / 配角：闻雪茶 / 龙套：邻居大妈；NPC：路人×3
```

> 用 `/` 分隔三个层级。若该场无配角则省略配角段，无龙套/NPC则省略该段。

### 2.12 分场判断逻辑

> 🔴 分场是**有成本的决策**，必须满足以下三个条件之一才允许分场：

1. **时间断裂**：场景间存在不可压缩的时间跳跃（非同一连续时间段）
2. **空间不可达**：角色无法在合理时间内从 A 地到达 B 地
3. **戏剧主体切换**：核心冲突/目标发生本质变化，且不在同一物理空间

**不满足任一条件 → 合并为同一场。** 每场 action ≥ 6，不足则合并到相邻场。跨场角色需有离场+入场衔接。

### 2.13 叙事段落演绎化

> 🔴 原文中无台词的叙事段落不得跳过，必须用动作行 + 字幕/旁白完整演绎。

- 逐段比对原文，确认所有叙事段落均已转化为可视化内容
- 无台词段落 → 用 `▲` 动作行呈现画面 + `旁白：` 或 `【字幕：】` 补充信息
- 🔴 禁止以"无台词"为由跳过原文段落

## 三、节奏规则

### 3.1 0帧起手·开场钩子

> 🔴 每集第一秒必须**声画同步开炸**——画面抓眼球的同时，角色直接开口。不铺垫、不过渡、不留空镜缓冲，0帧起手。

**核心逻辑**：短视频用户的滑走决策在 1–2 秒内完成。画面只能留住眼睛，声音才能留住注意力。开篇第一句话和第一帧画面共同构成"不可滑走"的双重钩子。

**开场钩子的双轨要求：**

- 🔴 **画面轨**：第一个动作行必须是高信息密度的视觉冲击（冲突现场/极端表情/反差场景/危机画面）
- 🔴 **声音轨**：第一句对白必须与画面同步，角色直接开口说出具有钩子力的台词——让观众听完这句话就想知道"后面怎么了"
- 🔴 **禁止空镜开场**：不得以空镜/环境建立/旁白铺垫作为第一行，必须直接进入角色行动与对白

**开场首句台词的钩子类型（任选其一）：**

| 类型 | 效果 | 示例 |
|------|------|------|
| **悬疑** | 抛出未知，制造信息缺口 | "这扇门后面，不是你想的那样。" |
| **欲望** | 角色目标直给，观众代入 | "今天，不是你死就是我活。" |
| **矛盾冲突** | 关系对撞，张力拉满 | "你说的每一个字，我都不信。" |
| **反转预告** | 颠覆预期，激发好奇 | "三天前，她还是我最好的朋友。" |
| **情绪炸弹** | 强烈情绪直击，引发共鸣 | "滚！我再也不想看见你！" |

**结构模板（场记格式）：**

```
{ep}-{scene} 时间 内/外 地点
人物：角色A、角色B
状态：角色A【状态1】、角色B【状态2】
▲高冲击画面 → 角色状态/动作
角色名（强情绪）：钩子台词（悬疑/欲望/矛盾/反转/情绪炸弹）
```

- 非默认造型只写在独立的 `状态：` 行
- 状态语法固定为 `角色【状态名】`，禁止圆括号状态和 `人物：角色【状态】`

### 3.2 节奏核心原则

- 🔴 **开场钩子**：遵循「0帧起手」规则，声画同步，角色直接开口，禁止空镜开场
- 🔴 **转折频率**：每 15 秒抛出一个转折/冲突/新信息
- 🔴 **场景节奏**：每场只解一问再抛一问；节奏明快，避免拖沓
- 🟡 **女频用词**：将女频小说的惯用词汇合理融入
- 🔴 **情绪峰谷**：60 秒内至少一次情绪高峰、一次情绪低谷
- 🔴 **集间钩子**：每集结尾悬念钉子与下集开场钩子必须衔接

## 四、去 AI 味七维度检查

> 从七个维度系统消除 AIGC 痕迹，每维度至少扫描一遍。修改原则：用具体的物理动作/微表情替换抽象描写，而非简单删除。

| 维度 | 检查要点 |
|------|---------|
| **结构规整** | 是否每场都是"起承转合"四段式？打破对称，允许不均匀节奏 |
| **表达重复** | 同一情绪/动作是否用了相同句式？替换为不同角度的具体描写 |
| **逻辑过度** | 是否每个决定都有过于完整的内心推理？删减显而易见的因果链 |
| **情感缺失** | 角色反应是否千篇一律（震惊→沉默→决心）？加入意外的、不合常理的真实反应 |
| **词汇单调** | "不禁""竟然""缓缓""默默""渐渐"等模板化副词？删除或换具体动作 |
| **描写空洞** | "眼中闪过一丝XX""嘴角微微上扬"等公式？换为角色独有的身体语言 |
| **对白雷同** | 遮住角色名能否区分谁在说话？每人需有独特语体和节奏 |

## 五、质量红线

- 🔴 **最小集长度**：每集不少于 300 字对话/自白/心声
- 🔴 **对白深度**：人物对白有深度和复杂性，避免直接或浅显的表达
- 🔴 **文风仿写**：紧密依循原作文风和情感基调
- 🔴 **口语化**：使用生活化词汇，加入语气词如"搞什么鬼""怕什么"
- 🔴 **禁止空话**：所有台词服务冲突或信息
- 🟡 **情绪对比**：通过对比不同情绪，让角色情感冲突更明显
