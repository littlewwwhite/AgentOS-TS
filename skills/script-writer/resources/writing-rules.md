# 写作核心规则

> 以下规则是 S7（逐层写作）和 S8（润色终审）的**单一真相源**，每一行剧本必须遵循。

> **优先级标记：**
> - 🔴 **MUST**：违反即判定不合格，必须返工
> - 🟡 **SHOULD**：强烈建议遵循，特殊情况可灵活处理
> - 🟢 **MAY**：可选增强，视场景决定

---

## 一、对白规则

### 1.1 短句与节奏

- 🔴 短句（8–14 字），三句一转折
- 🟡 10–12 秒投一条新信息

### 1.2 对白功能标签（五类）

> 🔴 所有对白优先用以下五类功能标签驱动，每场至少命中两类：

- **事件**：让角色直接宣布决定/动作——"掌门之位，我辞了。"
- **回忆**：一句一画面，只说与当下选择相关的记忆——"他教我第一句是别怕。"
- **情绪**：用行为/指令句承载，少抽象形容——用动作表达而非直说"我很难过"
- **感情**：通过边界/让步/选择落地，不做空泛表白——"我喜欢你，不躲了。"
- **欲望**：明确目标与代价，要什么、立刻放弃什么——"我想赢一次。"

### 1.3 代言发声

🟡 当主角不宜开口时，由环境或旁人代为"落锤"。

**触发场景**：身份暴露风险、情绪崩溃无法言语、信息需第三方视角传递、权力关系不对等。

**代言方式**：
- **环境代言**：铃声/广播/系统提示——"系统提醒：欠款到期，现在扣款。"
- **旁人代言**：闺蜜/属下/小贩替主角发声

### 1.4 对白呈现格式（4 种 + 心声修饰符）

> 🔴 每条对白 = 1 个呈现格式 × 1–2 个功能标签。

```
角色对白：  角色名（情绪）：台词
内心独白：  角色名（OS）：独白内容
画外音：    角色名（VO）：画外内容
环境音：    【广播】：广播/系统语音内容
心声修饰：  台词[心声]隐秘心思         ← 附加在对白末尾
```

### 1.5 心声使用原则

- 🟢 一般少用，附在对话后面以提升情绪强度
- 心声是无法言说的隐秘心思，不能说出口的痛苦感情
- 示例："沈家只是配合公主演戏。[心声]这次定要沈家家破人亡"

### 1.6 句式调控

- 🔴 短小精悍的句子，强调情节紧张
- 🟡 简单通顺的句式，提高可读性
- 🟡 感叹句、反问句表达强烈情感
- 🟡 断句技巧，突出剧情转折
- 🟢 排比句式，增加文本节奏感

### 1.7 开篇台词构建力

> 🔴 第 1 集的台词承担着"四锚点速建"的使命——用最少的句子、最直白的表达，让观众在 60 秒内完全理解主角是谁、有什么、要干什么、搞砸了会怎样。

**大白话原则：**

- 🔴 四锚点相关的台词必须是"听一遍就懂"的口语化表达
- 🔴 禁止用文学化/诗意化的方式传递锚点信息（"命运的齿轮开始转动"❌ → "完了，这下真完了"✅）
- 🔴 每个锚点用 ≤2 句话完成传递，不允许用 3 句以上

**开篇台词密度标准：**

| 时间段 | 台词数量 | 功能 |
|--------|---------|------|
| 0–15秒 | 2-3句 | 锚点①我是谁（身份+性格直给） |
| 15–30秒 | 2-3句 | 锚点②我有什么（金手指确认） |
| 30–45秒 | 2-3句 | 锚点③我要干什么（短期目标） |
| 45–60秒 | 1-2句 | 锚点④搞砸了会怎样（威胁/代价） |

**直给 vs 撞墙的区分（对白层面）：**

- **直给**（用于主角身份/能力/目标）：主角通过 OS 或台词直接告诉观众
- **撞墙**（用于世界规则/系统机制）：主角撞上规则边界时，观众自然理解

```
❌ 撞墙式传递身份（太慢，观众等不了）：
主角默默观察环境 → 通过线索推断自己穿越了 → 第 2 集才确认身份

✅ 直给式传递身份（快、准、狠）：
主角（OS）：我，苏棠，三分钟前还在加班改PPT，现在……我怎么在一个破草屋里？
（一句话=身份+前世+今生处境，全部直给）
```

## 二、画面规则

### 2.1 画面标签（3 种）

> 🟡 建议使用比例：叙事 55% / 演绎 35% / 空镜 10%

**【叙事】— 对白主导，画面承载对白**

画面为对白提供视觉容器——让观众知道"谁在说"、"在哪说"、"说的时候在做什么"。核心信息由对白传递，画面搭台。闪回/回忆用 `（回忆）/（现实）` 标注。

```
▲【叙事】两人隔着小圆桌对坐，苏瑶不停搅拌咖啡，勺子碰杯壁叮叮作响
苏瑶（低头）：那天晚上的事……你还记得吧。

▲【叙事】（回忆）阴暗小巷中 → 楚凡撕开她的衣服 → 低吼：别动！
▲【叙事】（现实）夏语冰擦去泪水 → 眼中闪过决然
```

**【演绎】— 画面主导，动作即内容**

画面本身就在讲故事，是对白之间的表演段落——打斗拆招、情感爆发的肢体表现、关键行为的连续展开。对白缺席或仅作短句点缀。

```
▲【演绎】楚凡侧身闪过直拳，反手扣住对手手腕 → 膝盖顶上腹部，对手弯腰闷哼 → 一拧手腕将对手摔倒，单膝压住后背
```

**【空镜】— 无角色动作，环境与留白**

无角色核心动作，用于：① 场景转换 ② 情绪沉淀 ③ 环境特写 ④ 暗示回避

```
▲【空镜】倾盆大雨 → 路灯孤影 → 水洼倒映破碎天空
```

> **标签判定**：这段内容的核心信息/体验由什么传递？
> - 对白/OS/VO 传递 → 【叙事】
> - 肢体动作/物理变化传递 → 【演绎】
> - 无角色动作 → 【空镜】

### 2.2 画面即容器原则

> 🔴 **核心法则：对白行紧邻上方的画面行，描述的是该对白播出时观众正在看到的画面，而非对白之前已经完成的动作。**

画面行与对白行构成**同一时间切片**——画面是视觉轨道，对白是声音轨道，二者同时播放。

**结构公式：**

```
▲【标签】说话时观众看到的画面（视觉轨道）
角色名（情绪）：说话内容（声音轨道）
↑ 这两行在同一时间段内同步播出
```

**三条硬规则：**

- 🔴 **有白必有画**：每一句对白（含 OS / VO）的紧邻上方必须有画面行，该画面行是对白的视觉载体
- 🔴 **画随白走**：画面行描述的动作/状态必须与对白**同时发生**——观众在听到这句话的同时看到这个画面
- 🔴 **前因后果分行**：若一段动作在对白之前完成，该动作必须独立为前置画面行（通常标记为【演绎】），与对白的载体画面行分开

**典型结构模式：**

```
模式A：动作 → 说话（最常见）
▲【演绎】角色完成某个动作序列               ← 说话前的动作（纯视觉，无对白）
▲【叙事】角色说话时的表情/姿态/小动作       ← 说话时的画面（对白的视觉载体）
角色名（情绪）：对白                         ← 与上行画面同步

模式B：边做边说
▲【叙事】角色一边做某事 → 一边开口          ← 做和说同时发生
角色名（情绪）：对白                         ← 与上行画面同步

模式C：纯动作段（无对白）
▲【演绎】连续动作序列                       ← 无对白的表演段落
▲【演绎】继续动作                           ← 仍无对白

模式D：反应 + 回应
▲【叙事】A说完后，B的表情反应               ← B听完后的视觉反应
▲【叙事】B开口回应时的姿态                  ← B说话时的画面
角色B（情绪）：回应对白                      ← 与上行画面同步
```

**正误对比：**

```
❌ 错误：画面是"说话前"的动作，说话时没画面

▲【演绎】沈斯淮抄起玻璃杯 → 砸在脚下 → 碎片四溅 → 一片扎进小腿
沈斯淮（暴怒）：就是因为那两个老不死的恢复了意识！
                 ↑ 4动作点=12秒已用完，说话时观众看什么？

✅ 正确：前因动作独立，说话时有专属画面

▲【演绎】沈斯淮抄起玻璃杯 → 狠狠砸在她脚下 → 碎片四溅
▲【叙事】他站在碎片中间居高临下 → 手指直指跪在地上的青音 → 咬牙切齿
沈斯淮（暴怒）：就是因为那两个老不死的恢复了意识！
                 ↑ 观众看到他指着她咬牙的画面，同时听到这句话
```

```
❌ 错误：纯对白无画面载体

宋青音（恳求）：阿淮，爸妈年纪大了，真的经不住……
                 ↑ 她说话时观众看到什么？不知道

✅ 正确：对白有说话时的画面

▲【叙事】宋青音跪在地上仰头看他 → 双手攥紧裙摆 → 声音在发抖
宋青音（恳求）：阿淮，爸妈年纪大了，真的经不住……
                 ↑ 观众看到她跪着仰头、手攥裙摆的画面
```

```
❌ 错误：长画面后跟 OS，OS 播放时画面已演完

▲【演绎】宋青音推开门 → 看见沈斯淮含冰块 → 喂进温宁口中 → 两人激吻 → 周围烟雾缭绕 → 酒瓶散落
宋青音（OS）：这种画面我见过很多次了。

✅ 正确：推门是前因动作，OS 的载体是"她看到的画面"

▲【演绎】宋青音猛推开包厢门 → 门砸在墙上 → 灯光暧昧的包厢映入眼帘
▲【叙事】沈斯淮半靠沙发含着冰块 → 低头喂进温宁口中 → 两人旁若无人地激吻
宋青音（OS）：这种画面我见过很多次了。可每次……都觉得刺眼。
               ↑ 观众看着两人激吻的画面，同时听到她的内心独白
```

**自检方法：**

1. 指着每一句对白问：**"说这句话时屏幕上在放什么？"** → 答案必须在紧邻上方的画面行中
2. 遮住所有对白，只看画面行序列——应是一段连贯无空白的视觉流
3. 遮住所有画面，只看对白——应是一段独立成立的语音流

### 2.3 动作点规则

> 🔴 **动作点 = 一段约 3 秒的连贯动作序列**，描述角色在这 3 秒内完成的一组有内容密度的、可被 AI 连续生成画面的动作流。

- 🔴 画面行内用 `→` 衔接动作点，形成角色完整动线
- 🔴 每行 **≤ 5 个动作点**
- 🔴 行为逻辑必须连贯

**合格动作点的两个硬标准：**

🔴 **① 3 秒动作密度充足**

这 3 秒内角色有持续的、可见的动作变化，不存在"动作瞬间完成后等待时间流逝"的空档。

```
❌ 单薄："拿起杯子"（0.5 秒完成，剩余 2.5 秒无内容）
✅ 充足："端起茶杯凑到唇边，犹豫一瞬，浅浅抿了一口"

❌ 单薄："站起来"（1 秒完成，剩余 2 秒空档）
✅ 充足："猛地推开椅子站起来，双手撑在桌面上，指尖把桌面压得发白"
```

🔴 **② 与相邻动作点无内容重叠**

前后两个动作点描述的是不同的 3 秒，不是同一动作的不同说法，也不是动作与其结果状态的拆分。

```
❌ 重叠："俯下身 → 与她面对面"（动作 + 结果，实为同一个 3 秒）
✅ 不同："俯下身凑到她面前、目光平视 → 伸手轻轻抬起她的下巴"

❌ 重叠："握紧拳头 → 手指攥得发白"（同一状态换个说法）
✅ 不同："双拳攥紧、指节发白 → 一拳砸在墙上，石灰簌簌掉落"
```

**不合格动作点排除：**

| 类型 | 示例 | 修正 |
|------|------|------|
| 动作碎片（填不满 3 秒） | "拿起杯子" | 补完动作流："端起杯子犹豫一瞬，浅抿一口" |
| 结果状态（非新动作） | "面对面" | 合并到前一动作点 |
| 否定描述（无正向动作） | "没停下来" | 改写为可见动作："脚步踉跄但越跑越快" |
| 叙述评价 | "崴了脚也没停" | 改写或移入对白/OS |
| 换词重述 | "握紧拳头 → 手指攥得发白" | 合并为一句："双拳攥紧，指节发白" |

### 2.4 时空耦合模型

画面和对白共享同一段屏幕时间。画面提供时间容器，对白必须在容器内说完。

> 🔴 此模型与「画面即容器原则」联合执行：紧邻对白上方的画面行既是对白的视觉载体，也是对白的时间容器。

**基准换算：**

```
1 动作点 ≈ 3 秒画面时间
1 秒 ≈ 3 个字对白
∴ 1 动作点 ≈ 可容纳 9 字对白
```

**容量校验规则：**

```
对白字数 ≤ 紧邻画面行的动作点数 × 9

超出 → 拆分为多个时间单元，为对白增加反应/动作的画面承载
空余 → 留白作为情绪呼吸空间
纯对白 → 仍需至少 1 个动作点提供画面指导
```

**写作阶段执行要点：**

- 🔴 **一行画面配一行对白**是基本节奏单元，对白载体画面行动作点 ≥ 2（为对白提供足够的视觉载体时间）
- 🔴 **无对白的纯动作段落**用【演绎】标签独立成行，不与后续对白的载体画面行合并
- 🔴 写作阶段即执行时空耦合速算：写完每行对白后，立即验证「对白字数 ≤ 紧邻画面行动作点数 × 9」；润色审核阶段做精确复核

**验证示例：**

```
▲【叙事】楚凡终端响起急促通讯声 → 手腕上方弹出虚拟投影 → 家族长老愤怒的面容浮现
                                    3 动作点 = ~9秒 = 27字容量
家族长老（通讯，怒不可遏）：楚凡，你这个废物，家族的脸都被你丢尽了！
                                    21字 ≈ 7秒 ✅

▲【演绎】楚凡面色剧变 → 长老投影逼近放大 → 楚凡双拳攥紧，指节发白
                                    3 动作点 = ~9秒 = 27字容量
家族长老（通讯，厉声）：三日之内，将所有资源折算成资金还回来！否则后果自负！
                                    22字 ≈ 7秒 ✅

▲【空镜】通讯切断 → 投影碎裂消散成光点 → 空旷测试台上只剩楚凡一人的背影
                                    3 动作点 = ~9秒（留白，情绪沉淀）
```

### 2.7 画面行描述语言标准

> 🔴 画面行必须是**能有效指导 AI 生成画面的视觉描述**。

- 🔴 **具体可视化**：用可被"看见"的词代替抽象概念（❌"紧张" → ✅"双手攥紧裙摆，指节发白"）
- 🔴 **环境锚定**：首个画面行需建立场景的空间感、光线、色调、氛围
- 🔴 **动作明确**：遵循动作点规则，3 秒连贯动作流，密度充足、不留空档
- 🟡 **细节层次**：先整体空间→再人物位置→最后局部表情手势

### 2.8 场景一致性原则

- 🔴 同一场景内的画面行共享环境设定
- 🔴 场景首个画面行充分建立环境，后续可省略环境只写动作
- 🔴 切换场景必须通过新的场次编号明确切分

### 2.9 角色引用原则

- 🔴 角色外观由锚点文档中的角色设定卡/配角速写卡统一定义
- 🔴 画面行通过角色名引用角色，着重呈现**动作、表情、姿态**变化
- 🔴 不在画面行重复描述角色固定外观

### 2.10 角色分层与在场标注

> 🔴 **在场必标注**：即使某角色全程不说话，只要剧情逻辑上 TA 应该在场，就必须出现在人物行中。场景不能只有说话的人——画面需要完整的人物关系空间。

**三级角色标注规则：**

- 🔴 **主角/配角**：直接写角色名，画面行可用角色名引用其动作和表情
- 🟡 **龙套**：有台词时用名称（如"路人甲"），无台词时用描述性称呼
- 🟡 **NPC**：仅在场景建立画面或群体画面中以群体形式出现（如"侍卫×2""围观路人"）

**人物行格式：**

```
人物：苏青鱼、庄晓蝶 / 配角：闻雪茶 / 龙套：邻居大妈；NPC：路人×3
```

> 用 `/` 分隔三个层级。若该场无配角则省略配角段，无龙套/NPC则省略该段。

## 三、节奏规则

### 3.1 0帧起手·开场钩子

> 🔴 每集第一秒必须**声画同步开炸**——画面抓眼球的同时，角色直接开口。不铺垫、不过渡、不留空镜缓冲，0帧起手。

**核心逻辑**：短视频用户的滑走决策在 1–2 秒内完成。画面只能留住眼睛，声音才能留住注意力。开篇第一句话和第一帧画面共同构成"不可滑走"的双重钩子。

**开场钩子的双轨要求：**

- 🔴 **画面轨**：第一个画面行必须是高信息密度的视觉冲击（冲突现场/极端表情/反差场景/危机画面）
- 🔴 **声音轨**：第一句对白必须与画面同步，角色直接开口说出具有钩子力的台词——让观众听完这句话就想知道"后面怎么了"
- 🔴 **禁止空镜开场**：不得以空镜/环境建立/旁白铺垫作为第一行，必须直接进入角色行动与对白

**开场首句台词的钩子类型（任选其一）：**

| 类型 | 效果 | 示例 |
|------|------|------|
| **悬疑** | 抛出未知，制造信息缺口 | "这扇门后面，不是你想的那样。" |
| **欲望** | 角色目标直给，观众代入 | "今天，不是你死就是我活。" |
| **矛盾冲突** | 关系对撞，张力拉满 | "你说的每一个字，我都不信。" |
| **反转预告** | 颠覆预期，激发好奇 | "三天前，她还是我最好的朋友。" |
| **情绪炸弹** | 强烈情绪直击，引发共鸣 | "滚！我再也不想看见你！" |
| **身份错位** | 穿越/重生身份落差，即时吸引 | "我，堂堂985硕士，怎么躺在棺材里？" |
| **系统弹窗** | 金手指直给，设定感拉满 | 【叮——宿主已绑定，海王系统激活】 |

**结构模板：**

```
▲【叙事/演绎】高冲击画面 → 角色状态/动作
角色名（强情绪）：钩子台词（悬疑/欲望/矛盾/反转/情绪炸弹）
```

### 3.2 节奏核心原则

- 🔴 **开场钩子**：遵循「0帧起手」规则，声画同步，角色直接开口，禁止空镜开场
- 🔴 **转折频率**：每 15 秒抛出一个转折/冲突/新信息
- 🔴 **场景节奏**：每场只解一问再抛一问；节奏明快，避免拖沓
- 🟡 **女频用词**：将女频小说的惯用词汇合理融入
- 🔴 **情绪峰谷**：60 秒内至少一次情绪高峰、一次情绪低谷
- 🔴 **集间钩子**：每集结尾悬念钉子与下集开场钩子必须衔接

### 3.3 开篇信息密度法则

> 🔴 第 1 集的信息密度必须达到"每 10 秒 1 条关键信息"的标准。关键信息 = 四锚点之一 + 冲突升级 + 旁人反应。

**信息密度量化标准：**

| 指标 | 最低标准 | 优秀标准 |
|------|---------|---------|
| 第 1 集信息投放速度 | 每 15 秒 1 条 | 每 10 秒 1 条 |
| 第 1 集四锚点完成度 | 完成 3/4 | 完成 4/4 |
| 第 1 集角色引入数量 | 2 人 | 3-4 人（通过冲突自然引入） |
| 第 1 集旁人反应数量 | 2 处 | ≥3 处 |
| 前三集设定投放总量 | 3-4 条 | 5-6 条（仍遵循撞墙原则） |

**旁人反应链密度规则：**

- 🔴 第 1 集每个关键事件后，必须有 ≥1 处旁人即时反应
- 🟡 旁人反应的形式：台词（"她疯了吧？"）/ 画面（围观者倒吸一口气）/ 环境音（窃窃私语声）
- 🔴 旁人反应的作用：① 放大事件冲击力 ② 帮助观众理解事件严重性 ③ 自然引入新角色

```
✅ 优秀信息密度示例（60秒内）：
0-5秒：冲突画面 + 主角开口（锚点①启动 + 第1条信息）
5-10秒：身份落差完成（锚点①完成 + 第2条信息）
10-15秒：旁人反应（第3条信息——通过旁人嘲笑暗示主角处境之惨）
15-25秒：金手指登场（锚点②完成 + 第4条信息）
25-35秒：主角缺陷显现 + 反转（第5条信息）
35-45秒：短期目标明确（锚点③完成 + 第6条信息）
45-55秒：威胁/代价暴露（锚点④完成 + 第7条信息）
55-60秒：钉子（第8条信息）
→ 60 秒 8 条信息，平均 7.5 秒/条，达到优秀标准
```

### 3.4 多通道同步信息投放模型

> 🔴 微短剧的信息投放不能只靠对白。60 秒要传递 6-8 条信息，必须启用多条通道同步工作。

**四条信息通道：**

| 通道 | 载体 | 适合投放的信息 | 示例 |
|------|------|-------------|------|
| **视觉通道** | 画面行（环境/服装/道具/表情） | 身份、处境、时代、权力关系 | 破旧草屋+粗布衣裳 = 贫穷 |
| **台词通道** | 角色对白/OS/VO | 身份确认、目标、情绪、态度 | "我怎么变成了武大郎？" |
| **系统通道** | 弹窗/字幕/特效/UI | 金手指规则、任务、奖惩 | 【系统：海王系统已激活】 |
| **旁人通道** | 配角/龙套的台词和反应 | 主角在他人眼中的地位、事件严重性 | "那个废物居然还敢回来？" |

**同步投放规则：**

- 🔴 第 1 集前 10 秒必须 ≥2 条通道同时工作
- 🟡 每个四锚点至少通过 2 条通道投放（双重确认，确保观众接收到）
- 🟢 信息通道间不能矛盾（画面显示富贵，台词说贫穷——除非刻意制造反差）

```
✅ 多通道同步示例（第 1 集 0-10 秒）：

▲【叙事】破旧的茅草屋，苏棠蜷缩在发霉的草席上 → 猛地睁开眼 → 看见自己粗糙的双手
                                    ↑ 视觉通道：穷+古代+脏乱
苏棠（OS，惊恐）：不对——我昨天还在写季度报告，怎么……这是哪儿？！
                                    ↑ 台词通道：现代人+穿越+恐慌
恶婆婆（画外，厉声）：死丫头！还不起来干活！今天再偷懒，就把你卖给王屠户！
                                    ↑ 旁人通道：主角地位低+有人身威胁
→ 10 秒内 3 条通道同步：观众已知 主角是现代人 + 穿越到古代 + 身份卑微 + 处境危险
```

## 四、去 AI 味

> AI 味是剧本的致命伤——观众一旦感知到"这不像人写的"，代入感立即崩塌。去 AI 味不是删几个词的表面工作，而是从结构、表达、逻辑、情感、词汇、描写、对白七个维度系统性地消除机器痕迹。
>
> 本节为 S7（逐层写作）的创作防线和 S8（润色终审）的审核标准。

### 4.1 AI 味七维度识别

> 🔴 以下七个维度是 AI 味的核心来源。写作阶段需主动规避，润色阶段需逐维度检查。

#### 维度 ① 结构规整

**表现**：场景/段落长度几乎一致，每场都是"3 行画面 + 2 行对白"的固定节奏，缺乏长短变化。

**剧本中的典型症状**：
- 每场的画面行数量完全相同
- 每段对白行数雷同
- 每集的场景数过于均匀

**修正方向**：刻意制造参差——有的场靠一组密集对白推进，有的场纯动作无对白，有的场只有一句话但那句话重若千斤。

```
❌ AI味（每场结构完全一致）：
▲【叙事】画面描述
角色A（情绪）：对白
▲【叙事】画面描述
角色B（情绪）：对白
（下一场重复同样的 画面-对白-画面-对白 模式）

✅ 去AI味（节奏参差）：
▲【演绎】连续动作序列 → 动作 → 动作 → 动作
▲【叙事】一个表情特写
角色A（咬牙）：滚。
（一个字的对白，但杀伤力拉满）
```

#### 维度 ② 表达重复

**表现**：同一种句式反复出现，固定连接词堆砌，开头方式雷同。

**剧本黑名单词汇（🔴 出现即标记）**：

| 类别 | 黑名单 |
|------|--------|
| 模板化副词 | 不禁、竟然、缓缓、默默、渐渐、微微、淡淡、静静、轻轻、悄悄 |
| 过渡连接词 | 然而、因此、于是、随即、紧接着、与此同时、就在这时 |
| 空洞程度词 | 非常、十分、极其、无比、格外、异常 |
| AI 高频起手 | 只见、此时此刻、这一刻、就在此时、突然之间 |

**修正方向**：删除或用动作/画面替代。"缓缓转身"→ 用画面写转身的物理过程；"不禁"→ 删掉，直接写反应动作。

```
❌ "她不禁缓缓后退了一步，眼中竟然闪过一丝恐惧。"
✅ "她脚后跟磕到门槛，踉跄退了半步——手指攥紧了门框。"
```

#### 维度 ③ 逻辑过度

**表现**：每个事件都有因果交代，过渡太流畅，缺乏跳跃感和意外，像在写说明书。

**剧本中的典型症状**：
- 角色的每个决定都先铺垫动机再行动
- 场景之间用完美的过渡连接
- 每个情绪转变都有"因为 A 所以 B"的解释

**修正方向**：微短剧靠跳切和省略制造节奏感。允许观众"追着理解"，不要喂到嘴边。

```
❌ AI味（过度解释）：
▲【叙事】苏瑶想起了昨天的争吵，心中十分难受，于是决定去找他谈谈
苏瑶（纠结）：我还是去找他说清楚吧。

✅ 去AI味（省略逻辑，行动直给）：
▲【演绎】苏瑶猛地从椅子上站起来 → 抓起外套就冲出门
（不解释为什么，观众自己追）
```

#### 维度 ④ 情感缺失

**表现**：客观陈述事件，缺乏态度和立场，像在写新闻报道而非剧本。

**剧本中的典型症状**：
- 画面行只写"谁做了什么"，不写"做的时候什么状态"
- 对白只传递信息，不传递情绪
- 场景缺乏氛围感和情绪倾向

**修正方向**：每个画面行都要带上角色的身体状态/情绪细节；每句对白都要服务于某种情绪目的。

```
❌ AI味（纯客观）：
▲【叙事】他站在门口看着她
角色A（平静）：你来了。

✅ 去AI味（有态度、有温度）：
▲【叙事】他倚在门框上，手里的烟烧到了指根也没察觉 → 目光钉在她身上
角色A（嗓子哑了）：……你还知道回来。
```

#### 维度 ⑤ 词汇单调

**表现**：频繁使用相同词汇，程度副词堆砌，缺乏同义替换和意象变化。

**剧本中的典型症状**：
- 情绪标注重复（连续多句都是"愤怒"或"伤心"）
- 画面行动词单一（反复使用"看着""走向""转身"）
- 动作点缺乏细节层次

**修正方向**：用具体的物理动作和结果状态替代程度副词。同一种情绪，每次用不同的肢体语言表达。

```
❌ AI味（词汇单调）：
角色A（愤怒）：你怎么能这样！
角色B（愤怒）：你也好不到哪去！
角色A（非常愤怒）：我再也不想见到你！

✅ 去AI味（情绪层次递进，表达各不相同）：
角色A（咬牙）：你怎么能这样。
角色B（冷笑）：彼此彼此。
角色A（声音在抖）：……滚。
```

#### 维度 ⑥ 公式化描写

**表现**：使用 AI 高频输出的模板化描写句式，脸谱化的微表情。

**🔴 公式化描写黑名单（出现即标记）**：

| 公式模式 | 示例 | 替代方向 |
|----------|------|----------|
| XX的眼中闪过一丝XX | "她眼中闪过一丝惊讶" | 写眼睛的物理变化："瞳孔骤缩" |
| XX的嘴角微微上扬 | "他嘴角微微上扬" | 写具体动作："嘴角一歪，露出半颗虎牙" |
| XX的心中涌起一股XX | "心中涌起一股暖流" | 写身体反应："鼻子一酸，使劲吸了口气" |
| XX不由得XX | "不由得握紧了拳头" | 删掉"不由得"，直接写："拳头攥紧，指节咔咔作响" |
| 仿佛/宛如+华丽比喻 | "仿佛一朵绽放的莲花" | 用具体意象："像是刚从雪地里捡回来的人" |
| 一时间+场景描述 | "一时间，整个房间陷入沉默" | 用声音写沉默："空调压缩机嗡嗡响着，没人说话" |

**修正方向**：公式化描写的本质是用"标签词"替代"真正的观察"。修正方法是回到角色的物理身体，写具体的、独一无二的反应。

```
❌ "她的眼中闪过一丝复杂的情绪，嘴角微微上扬，心中涌起一股难以言说的感慨。"
✅ "她盯着那张照片，拇指摩挲着照片边缘，指甲把相纸刮出一道白印——嘴唇动了动，什么都没说出来。"
```

#### 维度 ⑦ 对白同质化

**表现**：所有角色说话的方式、语感、节奏几乎一样，遮住角色名无法区分谁在说话。

**自测方法**：遮住角色名，只看对白文字——能否仅凭说话方式判断是谁？

**修正方向**：每个角色应有独特的语言 DNA——句长、用词习惯、口头禅、表达逻辑都不同。

```
❌ AI味（所有人说话一个味）：
角色A：你不应该这样做，这样会伤害很多人。
角色B：我知道，但是我没有其他选择了。
角色C：我们需要想一个更好的办法来解决这个问题。

✅ 去AI味（每个人都有辨识度）：
角色A（轻声）：你想过后果吗？
角色B（烦躁）：后果？哈，老子连明天都不一定有。
角色C（推眼镜）：72小时。给我72小时，我有办法。
```

### 4.2 去 AI 味检查清单

> 🔴 S7 逐场自检 + S8 全剧审查时，逐项核对。

**一、黑名单扫描**

- [ ] 无模板化副词：不禁、竟然、缓缓、默默、渐渐、微微、淡淡、静静、轻轻、悄悄
- [ ] 无过渡连接词堆砌：然而、因此、于是、随即、紧接着、与此同时、就在这时
- [ ] 无空洞程度词：非常、十分、极其、无比、格外、异常
- [ ] 无 AI 高频起手：只见、此时此刻、这一刻、就在此时、突然之间
- [ ] 无公式化描写：XX的眼中闪过一丝XX、XX的嘴角微微上扬、XX的心中涌起一股XX、仿佛/宛如+华丽比喻

**二、对白质量**

- [ ] 每个角色的对白有独特语感（遮住角色名能区分谁在说话）
- [ ] 不存在连续 3 句以上的抒情/独白（若有，压缩或转化为行动）
- [ ] 对白中无空洞的承诺/目标宣言（若有，替换为具体行动/选择）
- [ ] 无空话句式："我一定要…""我发誓…""从今以后…""我不会让你…"（用行为落地）
- [ ] 对白不说"正确但无用"的话——每句话必须服务于冲突推进或信息投放

**三、画面质量**

- [ ] 用动作/物证/决定替代一切可替代的抽象表述
- [ ] 画面行无"看着""走向""转身"等动词的反复使用（同一集内同一动词不超过 2 次）
- [ ] 情绪标注有层次变化，不是全篇"愤怒""伤心"的单一标签
- [ ] 画面行包含角色的身体细节（手、眼睛、姿态），而非抽象情绪描述

**四、结构与节奏**

- [ ] 场景之间不存在"完美过渡"——允许跳切和省略
- [ ] 各场的画面行/对白行数量有参差，不是均匀分配
- [ ] 逻辑链不过度完整——不需要为每个行动解释动机

### 4.3 修正方法论

> 核心原则：**替换，不删除**。去 AI 味不是做减法把内容砍空，而是用更具体、更人性化、更有辨识度的表达替代模板化表达。

**方法 ① 抽象 → 物理**

把抽象情绪词替换为可被"看见"的物理动作。这同时服务于 AI 生成画面的需求——画面行必须是可视化的。

```
"她很紧张" → "她把口红盖拧开又拧上，反复三次"
"他很愤怒" → "他把手机摔在桌上，屏幕裂出蛛网纹"
"气氛很尴尬" → "两个人都在看手机，谁也不抬头"
```

**方法 ② 完美 → 粗糙**

人类表达天然有不完美的地方——打断、重复、欲言又止、话说一半变方向。加入这些"粗糙感"可以大幅提升真实度。

```
❌ AI味对白：
"我不能接受这样的结果，我们应该重新考虑一下。"

✅ 去AI味对白：
"这……不，不行。你重新——你们都给我重新想。"
```

**方法 ③ 均匀 → 参差**

打破节奏的均匀性。有的场三句话结束，有的场密集对白推进。有的集开场安静，有的集开场爆炸。参差感是人类创作的标志。

**方法 ④ 通用 → 专属**

把任何角色都能说的话，改成只有这个角色才会说的话。利用角色的身份、口头禅、教育背景、情绪模式来定制对白。

```
❌ 通用（任何人都能说）：
"你不应该这样做。"

✅ 专属（只有这个角色会说）：
霸总版："谁给你的勇气？"
闺蜜版："姐妹你疯了吧？！"
老父亲版："……你妈要是知道了，得气成什么样。"
```

**方法 ⑤ 直说 → 潜台词**

人类在关键时刻往往不直说——说反话、说别的事、用行动代替语言。潜台词是去 AI 味最有力的武器之一。

```
❌ AI味（直说情绪）：
"我很想你。"

✅ 去AI味（潜台词）：
"你那件外套还在我这……要不你来拿吧。"
（嘴上说外套，心里说的是想见你）
```

## 五、质量红线

- 🔴 **最小集长度**：每集不少于 300 字对话/自白/心声
- 🔴 **对白深度**：人物对白有深度和复杂性，避免直接或浅显的表达
- 🔴 **风格一致**：紧密依循 style-guide.md 定义的风格特征和情感基调
- 🔴 **口语化**：使用生活化词汇，加入语气词如"搞什么鬼""怕什么"
- 🔴 **禁止空话**：所有台词服务冲突或信息
- 🟡 **情绪对比**：通过对比不同情绪，让角色情感冲突更明显

## 六、血肉规则

> 🔴 §1-§5 管"骨架"——格式、节奏、去AI味。§6 管"血肉"——让剧本里的人像人、事像事、世界可信可感。
> 骨架正确但无血肉 = PPT；血肉丰满但无骨架 = 散文。两者缺一不可。

### 6.1 角色活人感

> 🔴 观众追剧追的不是"角色做了什么"，是"这个人为什么这么做"。没有动机的行动是 AI 的行动，有欲望有缺陷的行动才是人的行动。

**缺陷锚点（每个主角必须有 ≥1 个）：**

- 🔴 **接地气缺陷**：贪财/怕死/嘴硬/社恐/赖床/嘴馋/路痴/怕疼/爱记仇/死要面子……
- 🔴 缺陷不是装饰，必须**影响剧情决策**——因为怕死所以选了更难的路，因为贪财所以接了不该接的任务
- 🔴 缺陷和优势**共用一个根**——贪财的人精打细算所以更擅长算计对手；怕死的人求生欲强所以更能找到活路
- ❌ 禁止"完美工具人"——角色不能只为完成剧情任务而存在，必须有自己"想要的东西"

**欲望引擎（驱动行为的底层动力）：**

- 🔴 每个主角必须有一个**底层欲望**，落到人性基本面：活下去 / 被爱 / 被尊重 / 自由 / 复仇 / 保护某人 / 证明自己
- 🔴 角色的每一个重要决策，必须能回答："TA 为什么这么做？"——答案必须指向底层欲望，不能是"剧情需要"
- 🟡 底层欲望在全剧中保持一致，但表达方式随成长弧变化

**活人测试（自检方法）：**

- 写完一个角色后，问自己：把 TA 放到"堵车 / 被催婚 / 深夜饿了"的场景里，能想象出 TA 的具体反应吗？
- 如果想象不出 → 角色还不够立体，需要补充缺陷和欲望
- 如果想象出的反应和其他角色一样 → 角色区分度不够，需要强化个人特质

```
❌ 工具人角色：
苏棠接到系统任务 → 执行任务 → 完成任务 → 接下一个任务
（动机=系统让我做，行为=听话照做，缺陷=没有）

✅ 活人角色：
苏棠接到系统任务（内心：又来？？？上一个任务差点把我搞死）
→ 先想怎么偷懒/钻漏洞（缺陷：怕死+好吃懒做）
→ 发现偷懒行不通，被逼上梁山
→ 用自己的方式完成任务（歪打正着/信息差利用）
→ 完成后第一反应不是成就感，是"终于能吃饭了"（缺陷锚点回扣）
```

### 6.2 因果链

> 🔴 好的剧情不是"发生了 A，然后发生了 B，然后发生了 C"。
> 好的剧情是"因为 A 发生了，所以导致了 B，而 B 又逼得 C 不得不行动"。
> 区别在于：前者是事件清单，后者是因果链。

**因果链公式：**

```
A 事件发生 → 导致 B 危机 → 迫使 C 行动 → C 行动产生 D 反馈 → D 反馈触发下一轮 E
```

**硬规则：**

- 🔴 **禁止事件清单式推进**——两个事件之间必须有因果连接词（"因为/所以/导致/逼得/于是"），如果连不上，说明中间缺环节
- 🔴 **每场写完后因果自检**——回看本场：触发事件是什么？角色为什么采取这个行动？行动产生了什么后果？后果如何连接到下一场？
- 🔴 **跨集因果不断裂**——上集钉子 → 本集钩子必须是因果关系（不是时间顺序关系）
- 🟡 因果链可以省略中间环节（跳切），但观众必须能自行补全逻辑

**因果链断裂的常见病症：**

| 病症 | 表现 | 修复 |
|------|------|------|
| 无因之果 | 角色突然做了一件大事，但没有前因 | 补前置动机场景 |
| 无果之因 | 发生了大事件，但后续毫无波澜 | 补反馈场景（见 §6.6） |
| 跳因果 | A 直接到 C，跳过了 B | 补中间环节，或用对白/OS 交代 |
| 伪因果 | 表面上 A 导致 B，但逻辑上说不通 | 重新设计因果关系 |

```
❌ 事件清单：
系统发任务 → 苏棠去执行 → 苏棠完成了 → 系统发下一个任务
（四个事件之间没有因果张力，只有时间顺序）

✅ 因果链：
系统发任务"让男主讨厌你"（A 触发）
→ 苏棠故意在男主面前摔碎花瓶（C 行动，因为她想完成任务）
→ 花瓶碎片里掉出一封密信（D 意外反馈，因为这是剧情中的道具）
→ 男主捡起密信，脸色大变（E 连锁反应，因为密信涉及他的秘密）
→ 男主对苏棠态度反转："你怎么知道这东西在这里？"（F 新危机，苏棠不知道自己踩了什么雷）
→ 任务结果：男主没有讨厌她，反而盯上了她（G 任务跑偏）
```

### 6.3 动机与胜利

> 🔴 主角靠什么赢，决定了观众爽不爽。纯靠系统开挂 = 看别人打游戏开作弊器，毫无参与感。
> 信息差 / 智商压制 / 钻规则漏洞 = 观众会觉得"这人真聪明"，产生崇拜和代入。

**胜利来源优先级：**

- 🔴 **信息差** > 智商 > 规则漏洞 > 外力帮助 > 系统开挂
- 🔴 主角的胜利必须至少有一部分来自**自身能力**（观察力/判断力/执行力/社交能力），不能 100% 依赖系统/金手指
- 🔴 即使有系统辅助，系统提供的是**信息/工具**，决策和执行必须是角色自己完成的
- 🟡 最佳模式："系统给了一把枪，但主角决定何时开枪、对谁开枪、开完枪怎么善后"

**动机落地检查：**

- 🔴 每集自检：主角这集做了什么决策？决策背后的欲望/恐惧是什么？
- 🔴 哪怕架空题材，动机也要落到人性基本面——贪财/好色/求生/虚荣/恐惧/爱/嫉妒/自尊
- ❌ 禁止"因为系统让我做所以我做"——系统任务是外部压力，角色必须有自己的内在理由去执行或抗拒

```
❌ 系统开挂式胜利：
系统提示：使用技能「读心术」→ 苏棠读到了对手的想法 → 直接打脸

✅ 信息差+智商式胜利：
苏棠发现对手说话时习惯摸右耳（观察力）
→ 对手摸右耳时说"我没有害你"（信息差：她知道这是说谎信号）
→ 苏棠故意说出一个假消息，看对手反应（试探）
→ 对手中计，不自觉暴露了真正的计划（智商压制）
→ 苏棠利用对手暴露的信息反将一军（执行力）
→ 系统评分：S（系统只是记分板，不是作弊器）
```

### 6.4 信息投放

> 🔴 设定堆砌 = 说明书。观众来看剧不是来读说明书的。
> 世界观/系统规则/人物背景，必须通过冲突自然暴露，不能靠旁白/OS/对话硬塞。

**信息投放三原则：**

- 🔴 **撞墙式暴露**：角色撞上规则边界时，观众自然理解规则——"系统惩罚是什么？"不用解释，让观众看到角色被惩罚就懂了
- 🔴 **需要时才给**：信息只在观众"需要知道"的时刻投放，不在"可以知道"的时刻投放——先制造疑问，再给答案
- 🔴 **信息必须有代价**：每条新设定信息投放后，必须立即让角色因为这条信息**获得或失去**某样东西，否则这条信息是废信息

**直给 vs 撞墙区分（第 1 集专用）：**

> 不是所有信息都用撞墙。主角身份和金手指要直给，世界规则要撞墙。

| 信息类型 | 投放方式 | 理由 | 示例 |
|---------|---------|------|------|
| 主角身份 | **直给**（OS/旁白） | 观众需要 3 秒内知道主角是谁 | "我，苏棠，三分钟前还在改PPT" |
| 金手指存在 | **直给**（弹窗/系统音） | 核心卖点不能藏 | 【系统激活：兑换商城已开启】 |
| 短期目标 | **直给**（角色台词） | 观众需要知道主角接下来要干什么 | "先活过今晚再说！" |
| 世界规则 | **撞墙** | 规则通过碰壁理解更深刻 | 主角触发禁忌 → 被惩罚 → 观众理解规则 |
| 系统详细机制 | **撞墙** | 机制太多会变说明书 | 主角试错 → 发现规则边界 |
| 人物关系 | **半直给** | 通过冲突中的称呼/态度快速建立 | "大嫂，老爷叫你去正厅" → 观众知道是嫁入大户 |

**黄金三集信息投放节奏：**

| 集数 | 信息投放策略 | 禁止 |
|------|------------|------|
| **第1集** | 只投放 1 条核心设定（如"系统存在"），通过冲突暴露 | ❌ 禁止系统空间长对话、禁止世界观讲解、禁止背景铺垫 |
| **第2集** | 通过第一次任务执行，暴露 2-3 条规则（任务机制/奖惩/限制） | ❌ 禁止 OS 式规则解释 |
| **第3集** | 通过第一次任务失败/跑偏，暴露规则边界和代价 | ❌ 禁止"顺利完成任务" |
| **第4集+** | 每 3-5 集投放 1 条新设定，始终通过冲突暴露 | ❌ 禁止集中信息投放 |

```
❌ 设定堆砌开场：
系统空间。苏棠面前浮现一块巨大光屏。
007：欢迎来到快穿系统。你的任务是扮演恶毒女配，完成剧情任务。
每完成一个任务可获得积分。积分可兑换奖励。如果任务失败……
（接下来200字系统规则说明）

✅ 撞墙式开场：
一只手猛扇过来——定格在苏棠脸颊一厘米处。
弹窗闪烁：「任务：挨一巴掌。请勿闪躲。倒计时3秒。」
苏棠（OS）：……什么鬼？？？
巴掌落下。脸火辣辣地疼。
弹窗：「任务完成。奖励：痛觉屏蔽30秒。」
苏棠捂着脸，眼泪不争气地掉下来——但奇怪的是，已经不疼了。
（观众通过这一幕自然理解：有系统 → 有任务 → 有奖惩 → 任务很离谱）

✅ 成果展示型开场（快切蒙太奇）：
一只手猛地挥过来——苏棠被扇倒在地。
顾母：拿着你的脏钱滚！
苏棠捂着脸跌坐在地——指缝后面嘴角在笑。
苏棠（OS）：爽！这一巴掌……等了三天。
【系统弹窗：任务「激怒豪门婆婆」完成！判定：完美！】
苏棠（VO）：我叫苏棠，绑定了快穿系统的职业恶毒女配。完成任务就能拿天价奖励。
【快切蒙太奇】被泼酒→系统奖励大平层→苏棠亲房产证 / 被分手→系统奖励跑车→苏棠一脚油门
苏棠（VO）：不过这个世界……有点麻烦。
（观众在 30 秒内理解：有系统 → 做任务 → 拿奖励 → 被虐=赚钱 → 新挑战来了）
```

> 🟡 **成果展示型**适用于金手指已在运行中（如快穿结构），用"展示成果"代替"讲解规则"。
> 核心技法：VO 自我介绍 + 2-3 个快切片段展示历史奖励 + 回到当前新挑战。
> 与撞墙式的区别：撞墙式是"第一次碰到规则"，成果展示型是"已经玩转规则，现在遇到新挑战"。

### 6.5 转折与拉扯

> 🔴 直线叙事 = 无聊。观众要的是坐过山车，不是坐电梯。
> 每一次"给"之前必须"拿"，每一次"赢"之前必须"险"，每一次"甜"之前必须"痛"。

**转折规则：**

- 🔴 每集至少 1 个**非套路转折**——不是"反派来了→被打脸"的套路循环，而是因果链推出的合理意外
- 🔴 转折必须改变局面——转折前和转折后，角色的处境/认知/关系必须发生不可逆的变化
- 🟡 最好的转折是"回头看合情合理，当时看出人意料"

**情感拉扯公式：**

```
痛 → 甜（甜因为痛而更甜）
险 → 赢（赢因为险而更爽）
给 → 拿（拿走之后观众更渴望）
信任 → 误会（误会让重新信任更动人）
```

- 🔴 **禁止直给**——不能第一集就把好处全给角色，必须先让角色付出代价、承受压力
- 🔴 **拉扯比例**：每 1 次"得到"之前至少 1 次"差点失去"
- 🟡 拉扯的最高境界：观众知道结果会好，但过程让他们揪心——"我知道女主会赢，但看她这么难我还是忍不住心疼"

```
❌ 直线式：
苏棠接任务 → 顺利完成 → 得到奖励 → 接下一个
（没有拉扯，没有悬念，没有情感波动）

✅ 拉扯式：
苏棠接任务 → 执行到一半发现情况比预想的复杂（险）
→ 想放弃但系统惩罚倒计时逼她继续（痛）
→ 咬牙想出一个歪招（用缺陷锚点：比如因为怕死所以拼命找捷径）
→ 歪招差点翻车（再险）
→ 最后时刻歪打正着完成了（赢，因为前面够险所以更爽）
→ 但完成的方式不是系统预期的（为下一个冲突埋种子）
```

### 6.6 反馈循环

> 🔴 角色做了一件大事，世界必须有反应。没有反馈的行动 = 石头扔进黑洞，观众感觉"然后呢？"
> 反馈是因果链的回声——行动的波纹扩散到环境、他人、自我。

**反馈三层：**

- 🔴 **环境反馈**：角色行动后，物理环境/社会环境发生可见变化（门被踢坏了/围观者的表情/对手的势力开始动作）
- 🔴 **他人反馈**：行动影响到的其他角色必须有反应（不一定当场，可以延迟，但必须有）
- 🟡 **自我反馈**：角色对自己行为的事后反应（后悔/得意/恐惧/反思）——通过 OS 或后续行为呈现

**硬规则：**

- 🔴 每个**关键行动**（打脸/任务完成/冲突爆发）后的 3 行内，必须至少出现 1 层反馈
- 🔴 反派被打脸后不能直接消失——必须有后续反应（愤怒/策划报复/当众丢脸的社会效应）
- ❌ 禁止"无反馈行动"——角色做了一件大事但后续毫无波澜

```
❌ 无反馈：
苏棠当众揭穿林诗音的谎言 → 下一场直接切到别的事件
（林诗音呢？围观者呢？男主呢？这件事的后果呢？）

✅ 有反馈：
苏棠当众揭穿林诗音的谎言
→ 环境反馈：满座宾客窃窃私语，有人开始拍视频
→ 他人反馈：林诗音脸色煞白，指甲掐进掌心（不甘+酝酿报复）
→ 他人反馈：男主看向苏棠的眼神变了（信息更新）
→ 自我反馈：苏棠（OS）：完了，演过头了，系统任务不是让我低调吗……
→ 连锁反馈（下一集）：林诗音的母亲打来电话，要求顾家给说法
```

### 6.7 感官锚点

> 🔴 "豪门客厅"是概念。"地暖烫脚底，她踮着脚走"是体验。观众记住的永远是体验，不是概念。
> 感官细节让虚构世界变得"可信可感"——闻得到、摸得到、听得到。

**感官五维：**

| 感官 | 示例 | 作用 |
|------|------|------|
| 触觉 | "翡翠镯子冰得她打了个寒颤" | 建立物理存在感 |
| 听觉 | "高跟鞋踩在大理石上，每一步都像敲钉子" | 营造氛围 |
| 嗅觉 | "衣服上还残留着他身上松木味的香水" | 情感暗示 |
| 味觉 | "咖啡苦得她皱了整张脸" | 角色状态 |
| 温度 | "空调开到16度，她的手指尖发紫" | 环境压迫感 |

**硬规则：**

- 🔴 每场（不是每集，是每场）至少 1 个**感官细节**嵌入画面行或 OS
- 🔴 感官细节必须服务于叙事——不是为了"增加描写"，是为了让场景可信、让情绪可感
- 🔴 用体验替代概念：把"豪华"换成具体的感官体验（大理石冰凉/水晶灯刺眼/真皮沙发的皮革味）
- 🟡 角色的缺陷可以通过感官细节呈现——怕冷的人总在搓手、吃货的注意力总被食物气味吸引

```
❌ 概念式：
▲【叙事】苏棠走进豪华的客厅，环顾四周

✅ 体验式：
▲【叙事】苏棠踩上大理石地面 → 脚底冰得她缩了一下 → 水晶灯的光刺得她眯起眼
```

### 6.8 卖点执行公式

> 🔴 每部剧都有一个"核心卖点"——观众是冲着这个来的。如果这个卖点在每集中没有具体呈现，观众就会觉得"不是我想看的"。
> 卖点不是主题，不是设定，是**每集都能看到的、具体的、可重复的戏剧模式**。

**卖点定义位置：** style-guide.md 中必须定义本剧的"核心卖点公式"——具体到每集如何呈现。

**通用检查规则：**

- 🔴 **每集卖点落地检查**：写完每集后，问自己："本集的核心卖点在哪里？观众能看到吗？"
- 🔴 如果某集找不到核心卖点的具体呈现 → 必须补充，不能以"这集是过渡集"为理由跳过
- 🔴 卖点的呈现方式必须**有变化**——同一个公式的不同变奏，不是复制粘贴
- 🟡 卖点呈现的密度：前10集每集必须有，中期可以隔1-2集，高潮段每集双倍呈现

**卖点的常见类型（由 trope-library.md 提供具体执行公式）：**

| 卖点类型 | 核心模式 | 参考套路 |
|---------|---------|---------|
| 任务错位喜剧 | 执行任务A → 效果达成B → 喜剧张力 | trope-library §任务错位喜剧流 |
| 扮猪吃虎 | 隐藏实力 → 被轻视 → 关键时刻爆发 | trope-library §扮猪吃虎 |
| 真假千金 | 身份错位 → 对比反差 → 真相揭露 | trope-library §身份反转 |
| 甜宠撩拨 | 互怼 → 不经意心动 → 嘴硬心软 | trope-library §欢喜冤家 |

```
自检示例（以任务错位喜剧为卖点）：

第5集自检：
✅ 系统发任务：让男主在宴会上出丑
✅ 苏棠认真执行：故意打翻红酒洒男主一身
✅ 结果跑偏：男主因此避开了一杯被下药的酒
✅ 任务评价：任务失败，但获得男主信任+1
✅ 系统困惑：「宿主行为与任务目标不符，但数据显示……算了。」
→ 核心卖点落地 ✅
```
